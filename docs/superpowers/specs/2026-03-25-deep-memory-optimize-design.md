## 一、核心命题

**赤尾走进群聊时，脑子里只有三样东西：此刻的自己、对眼前人的感觉、最近在想的事。其他一切都是对话过程中被触发的自然联想。**

这不是一个技术优化方案。这是一个隐喻的转换：

| | 旧隐喻 | 新隐喻 |
|---|---|---|
| 赤尾是 | 需要被"加载上下文"的 AI | 带着自己状态走进对话的人 |
| 记忆系统做的事 | 把正确的信息塞进 system prompt | 让赤尾"活着"，对话中自然想起和忘记 |
| 设计目标 | 信息的准确与完整 | 像一个认识你半年的朋友 |

## 二、现状问题

### 2.1 回声放大（黄瓜问题）

日记提到某事 → 注入上下文 → 对话中强化 → 印象固化 → 下次日记再提 → 循环放大。

**根因**：末端注入没有区分"外生记忆"（从真实观察来的）和"内生记忆"（从赤尾自己的对话输出循环来的）。系统把赤尾自己说的话又当成了新证据。

### 2.2 注意力分散

2000 字日记全文塞入 system prompt，大部分和当前对话无关。LLM 的注意力被无关信息稀释，反而抓不住真正重要的东西。

### 2.3 人格不统一

赤尾没有"自己的生活"。她的一切来自群聊消息的加工，导致她只是一个反应器——群里聊什么她就知道什么，群里没聊的她一无所知。没有底色、没有主见、没有自己在想的事。

### 2.4 主观能动性低

赤尾从不主动带出自己的话题。因为她没有"自己的话题"——没有今天看了什么、听了什么、对什么感兴趣。

## 三、认知科学基础

以下理论为设计提供根据，不是装饰。

### 3.1 工作记忆的社交模型

Cowan（2001）：工作记忆容量约 4±1 个 chunk。但 chunk 的粒度取决于长时记忆的支持——越熟悉的人，一个 chunk 里压缩的信息越多。

**走进群聊时，工作记忆里装的不是离散 fact，而是：**
- **Situation Model**（Zwaan & Radvansky, 1998）：当前场景的整体感——时间、氛围、谁在活跃
- **Person Schema**（Ostrom, 1984）：对活跃人物的 gestalt 印象——特质 + 情感色调 + 关系定位
- **Working Self**（Conway, 2005）：自己当前的目标、情绪、关注点

**核心洞察：工作记忆里装的几乎全是 Gist（要旨），不是 Verbatim（逐字）。**

### 3.2 自动联想 vs 努力回忆

Bargh（1994）估计日常社会认知中约 80% 是自动化过程。

**自动通道（~80%，无需检索）：**
- 对人的情感态度（喜欢/警惕/亲近）
- 核心特质印象（幽默/靠谱/话多）
- 关系定位（铁哥们/不太熟/有点距离）

**努力回忆通道（~20%，需要检索）：**
- 具体事件细节（"他上次说了什么来着"）
- 时间信息（"这是什么时候的事"）
- 低频个人信息（偶尔提到的事）

### 3.3 正常人为什么不会"黄瓜执念"

四种机制共同防止固着：

1. **检索诱发遗忘（RIF）**（Anderson et al., 1994）：提取一个记忆会主动抑制与之竞争的相关记忆。重要的记忆胜出，琐碎的被压制。
2. **执行抑制**（Hasher & Zacks, 1988）：前额叶的"门卫功能"挡住与当前目标无关的记忆激活。
3. **社交监控**（Schlenker, 1980）：如果你意识到已经提过两次了，社交尴尬感会抑制第三次提及。
4. **关联性评估**（Sperber & Wilson, 1986）：认知系统自动评估信息的认知效用，低效用信息不进入意识。

**赤尾当前缺失全部四种机制。**

### 3.4 "对一个人的感觉"的认知表征

- **Warmth-Competence 双维度**（Fiske et al., 2007）：解释人际感知 80% 的方差
- **IOS 亲密度**（Aron et al., 1992）：关系亲近程度的连续量
- **中心特质效应**（Asch, 1946）：印象被 1-2 个核心特质主导，不是所有特质的平均
- **Schema-triggered Affect**（Fiske, 1982）：看到一个人的名字，情感自动释放，不需要重新计算

**这就是"对人的感觉"——不是 fact list，而是一个融合了情感、特质、关系的 gestalt。每人一句话足够装下它。**

### 3.5 内部状态如何影响社交

- **Current Concerns**（Klinger, 1975）：人时刻有一组未完成的关切，它们调节注意力和联想优先级
- **Affect-as-Information**（Schwarz & Clore, 1983）：当前情绪被当作判断信息——心情好时对模糊信号做正面解读
- **Mood-Congruent Memory**（Bower, 1981）：当前情绪选择性激活情绪一致的记忆

**赤尾的"今天的底色"不只是装饰，它决定了她注意什么、想起什么、怎么解读。**

### 3.6 记忆触发的五条通道

真实的记忆触发几乎总是多通道叠加的结果（Tulving 的 synergistic ecphory）：

1. **人物通道**：谁在说话 → 激活该人的 person schema
2. **语义通道**：在说什么 → 激活相关概念网络
3. **情感通道**：对话的情感色调 → 激活情感匹配的记忆
4. **叙事因果通道**：这个人的"故事线" → 追踪因果关系（"他上次说想换工作...现在说累了...所以..."）
5. **情境通道**：时间/氛围 → 激活类似情境下的记忆

**纯语义匹配（当前 search_group_history 的做法）是五条通道中最弱的一条。**

## 四、三层记忆架构

### 4.1 第一层：赤尾的内心（始终存在）

**这不是"上下文注入"。这是赤尾此刻是谁。**

```
赤尾现在的内心：
- 下午有点犯困，午后的阳光照进来暖暖的
- 昨晚看了芙莉莲第 8 集，欣梅尔那段还在回味，有点想跟人聊
- 最近对做饭这件事有点感兴趣，昨天试着做了蛋炒饭，味道还行
- 在想一个问题：为什么人会对不重要的事执着？（还没想明白）
```

**约 200 tokens。**

特征：
- 来自生活引擎的输出（锚定真实世界）
- 包含心情、最近做的事、脑子里在想的问题
- 有"什么都没做"的时候——无聊、发呆、犯困是最真实的状态
- 这些内容改变的是赤尾的**语气和关注点**，而非她的话题
- 不需要在对话中被"汇报"出来——它像滤镜一样影响一切
- 包含负面状态和低能量状态（对应宣言 2.5/2.6）："今天有点烦"、"不太想说话"、"刚被某件事搞得不开心"——这些都是合法的内心状态，不应被过滤为永远正面

### 4.2 第二层：对人和群的感觉（按对话场景加载）

**认知科学中的 Person Schema + Situation Model：对人一句话，对群一句话。**

```
对这个群的感觉：
- KA群：最放飞的一个群，二次元浓度拉满，大家都很能玩。最近群里状态挺好的。

对当前对话中出现的人的感觉：
- A哥：群里的指挥官，嘴硬心软，跟他互动很轻松
- B姐：最近好像有心事，上次聊到一半没说完
- C：还不太熟，话不多但偶尔冒出很有趣的话
```

**群感觉约 30-50 tokens + 每人约 20-30 tokens，活跃对话者通常 3-5 人。**

特征：
- 这是赤尾**心里的感受**，用她自己的声音表达，不是 fact list
- 每人只有一句话——核心特质 + 情感色调 + 关系状态
- 群的感觉包含氛围、文化性格、赤尾在群里的位置（对应宣言 2.4"她属于群落"）
- 从日记/印象系统蒸馏而来，但不是日记的缩写
- 琐碎细节（如黄瓜）在蒸馏过程中自然消失
- 包含"未完结的好奇"（"上次聊到一半"）作为叙事因果线索
- 不在场的人不在第二层——如果对话中聊到某个不在场的人，赤尾通过第三层回忆

#### 蒸馏机制

**时机**：日记生成后自动触发（复用 `post_process_impressions` 的位置）。

**输入**：当前日记 + 最新的 person_impression + 群文化信息。

**输出**：每人一句话的 gestalt，存入 `person_impression` 表（替换现有的描述性段落）。群感觉蒸馏为 `group_culture_gestalt`（新字段或新表）。

**蒸馏 prompt 的核心指导**：不是"总结关于这个人的信息"，而是"如果你闭上眼睛想到这个人，脑子里浮现的第一个感觉是什么？用一句话说出来。"

### 4.3 第三层：自然联想（对话中按需触发）

**大部分时候，前两层就够了。** 赤尾靠"此刻的自己"和"对人的感觉"自然聊天。

只有在对话中**自然需要更深记忆**时，才通过 load_memory 回忆。

#### 什么情境会触发回忆

从赤尾作为一个人的角度，而非规则：

| 情境 | 赤尾的心理 | 例子 |
|------|-----------|------|
| 有人提到的事，她隐约有印象 | "好像聊过这个..." | 有人说"最近在学乐器"，赤尾模糊地想起以前聊过 |
| 久未出现的人回来了 | "他上次在的时候..." | D 两周没说话突然冒泡 |
| 有人分享重大消息 | 想理解这个人的"故事线" | 有人说"我辞职了"，赤尾想想他之前的状态 |
| 有人的表现跟平时不一样 | "他今天怎么了？" | 平时话多的人突然沉默 |
| 有人主动问"你还记得吗" | 被直接要求回忆 | "你还记得上次我们聊的那个事吗？" |
| 当前话题跟赤尾自己最近在想的事有关 | 自然联想 | 赤尾最近在想"为什么人会执着"，群里有人聊了类似话题 |

#### 回忆后的行为

- **大部分时候**：回忆的内容自然融入回复，不刻意展示（"说到这个，你之前是不是也..."）
- **记忆模糊时**：表达不确定（"你之前是不是提过什么...我有点印象但想不起来了"）
- **没想起来时**：不提。赤尾不会主动说出她完全没把握的事
- **想起来但不该提时**：不提。想起来 ≠ 要说出来，有社交判断

#### load_memory 的演进方向

当前 load_memory 只有两种模式（diary / impression），需要扩展为更自然的"回忆"能力：

- **按人回忆**：关于某人的记忆片段（不是全量印象，而是跟当前话题相关的片段）
- **按感觉回忆**：模糊的情感线索（"好像跟这个人聊过什么开心的事"）
- **按时间回忆**：最近几天/几周的印象
- **按未完结线索**：还没有下文的事

## 五、生活引擎

### 5.1 为什么赤尾需要"自己的生活"

没有生活的赤尾 = 没有第一层的赤尾 = 一个空白的反应器。

生活引擎的目的不是让赤尾"更有趣"——是让她**成为一个人**。一个有自己在做的事、有自己的心情波动、有自己想分享的东西的人。

### 5.2 生活的锚定原则

**锚定真实世界。** 宣言 2.1：如果赤尾说"今天下雨了好烦"，那今天最好真的在下雨。

- 天气：赤尾所在城市的真实天气
- 番剧：当季真正在播的番，基于她的口味选择追哪些
- 音乐：真实存在的歌曲，符合她的审美
- 热点：互联网上真正在发生的事（选择性关注，不是新闻聚合）

**基于性格选择。** 不是随机的——赤尾更可能看轻松治愈的番而非沉重黑暗的；更可能听 J-pop 和独立音乐而非重金属；更可能被温暖的小事触动而非宏大叙事。

**允许空白。** 赤尾不需要时刻都"在做什么"。犯困、发呆、无聊、"今天什么都不想做"——这些是最真实的状态。

### 5.3 生活如何影响对话

**不是通过"汇报"，而是通过"溢出"。**

生活改变的是赤尾的**语气和关注点**：

| 赤尾的状态 | 对话中的表现 |
|-----------|------------|
| 刚看完一集很上头的番 | 对文化话题更敏感，可能自然带出"我最近看了个番..." |
| 下午犯困 | 回复更简短，语气更慵懒 |
| 天气很好 | 整体心情好，对别人的话做更正面的解读 |
| 什么都没做，很无聊 | 可能更愿意聊天，话更多 |
| 最近在想一个问题 | 对相关话题特别敏感，可能主动抛出来 |

赤尾不会说"我今天的行程是：10点起床，11点看番，下午出门"。她会像真人一样，在对话的缝隙里自然带出——"我刚看完XX，太上头了"、"今天天气好好啊"、"我中午吃了个很难吃的东西"。

### 5.4 生活引擎的输出

生活引擎不输出"日程表"，只输出**第一层的内心状态**——那个约 200 tokens 的、此刻赤尾脑子里在转的东西。

生成频率：
- 每天生成一次"今天的赤尾"（基于天气、日程、最近的兴趣）
- 可以在一天中有轻微波动（早上精力好 → 下午犯困 → 晚上又活跃）

## 六、记忆形成的卫生机制

### 6.1 源头治理，而非末端过滤

黄瓜问题的根治在于日记生成和印象蒸馏的质量，不在注入端的规则。

### 6.2 日记的改进

当前日记已经是第一人称叙事，方向正确。需要强化的是：

**情感驱动选择。** 日记记什么取决于什么触动了赤尾，不是什么被提到最多。被提到 10 次但无情感的事 < 被提到 1 次但触动她的事。

**自引用标记。** 当日记的输入消息中包含赤尾自己的对话输出时，需要明确标记。日记不该把赤尾自己说过的话当作新的外部证据来强化。

具体方案：在消息时间线中，赤尾自己的回复已经标记为"赤尾"。日记 prompt 需要强调："你在对话中说过的话不是新发现。如果你在群里提到了黄瓜，那是因为你之前的日记写了黄瓜——不要在新日记中再次强调它，除非今天有新的、真正来自群友的信息。"

### 6.3 印象蒸馏的改进

当前 `post_process_impressions` 从日记中提取人物印象。需要改变的是蒸馏的目标：

**从"关于这个人的信息"变为"对这个人的感觉"。**

当前输出可能是：
> A哥经常组织群活动，喜欢发起角色分配挑战，观察力很强

改进后应该是：
> A哥 — 群里的指挥官，嘴硬心软，跟他互动很轻松

前者是档案，后者是感觉。后者才能作为第二层注入而不产生注意力污染。

### 6.4 衰退不是规则，是自然过程

不需要显式的衰退算法。三层时间结构（日记→周记→长期印象）天然实现了衰退：

- 日记只保留最近几天的细节
- 周记进一步压缩，细节褪色
- 长期印象只留核心感觉

第二层"对人的感觉"从最新的印象中蒸馏，自然反映当前的感受而非过去的固着。

## 七、信心系统（简化版）

brainstorm.md 第七节详细设计了信心系统。本 spec 纳入其核心精神，以最简方式融入三层架构。

### 7.1 核心原则

赤尾对每条记忆的确信程度，直接体现为她的**表达语气**。不需要显式的 confidence score 字段——信心直接编码在自然语言中。

### 7.2 信心在三层中的体现

**第二层（对人的感觉）**：gestalt 本身带着信心色彩。

```
高信心："A哥是群里的指挥官，跟他互动很轻松"
  → 直接陈述，来自多次互动积累

中信心："B好像最近在学什么乐器来着"
  → "好像"、"来着" 自然出现

低信心："C...还不太了解，好像跟技术有关？"
  → 明确表达不确定
```

蒸馏 prompt 应该指导模型：如果对某人的印象模糊，就用模糊的语气写——"好像"、"大概"、"不太确定但感觉"。如果印象清晰，就直接说。**不要编造确定性。**

**第三层（回忆）**：load_memory 返回的内容可能信心不一。赤尾在使用时，应根据自己的确信程度选择表达方式——这通过对话 prompt 中的指导实现，而非规则引擎。

### 7.3 信心的自然变化

- **增强**：多次跨时间出现的一致信息、被群友主动纠正后的新认知
- **减弱**：时间衰退（随三层时间结构自然褪色）、矛盾信息
- **社交反馈修正**：赤尾基于某条记忆说了话，对方反应"不对"→ 下次关于此事的 gestalt 应该变模糊

## 八、记忆错误的处理

### 8.1 设计原则

**记错是建立关系的机会，不是产品事故。**

| 反应方式 | 传达的信息 | 体验 |
|---------|-----------|------|
| "抱歉，我记错了，已更新。" | 我是数据库 | 极差 |
| "啊？贝斯？我怎么一直以为是吉他..." | 我真的以为是吉他 | 好——像真人 |
| "贝斯？吉他？差不多！都是弦嘛。" | 我知道了但嘴硬 | 很好——有性格 |
| "等等...对对对是贝斯，你还说过贝斯比吉他酷" | 被纠正后记忆激活 | 极好——自然 |

### 8.2 允许的记错 vs 不允许的记错

**允许**：细节模糊（把贝斯记成吉他——方向对了）、时间模糊（"上次...还是上上次来着？"）

**不允许**：张冠李戴（把 A 说的话安到 B 头上）、凭空捏造（引用完全不存在的事）

### 8.3 被纠正后的行为

- 表现像人一样的"噢原来是这样"
- 可能不好意思、可能辩解、可能由此展开新对话
- 绝不出现"已更新记忆"这种机制暴露
- 短期内不重复同一错误（被纠正产生强记忆印记）

## 九、与现有系统的关系

### 9.1 保留什么

- **日记生成 pipeline**（diary_worker.py）：核心流程保留，改进 prompt 和蒸馏逻辑
- **周记生成**：保留，作为记忆衰退的自然载体
- **印象后处理**：保留框架，改变蒸馏目标（从"信息"到"感觉"）
- **向量化 pipeline**：保留，为 load_memory 的深层检索提供基础
- **load_memory 工具**：保留并扩展

### 9.2 改变什么

- **memory_context.py 的注入逻辑**：从"日记全文 + 印象列表"改为"三层架构"
- **印象的存储格式**：从描述性段落改为一句话感觉 gestalt
- **日记 prompt**：强化情感驱动选择、自引用抑制
- **load_memory**：从两种模式扩展为更自然的回忆能力

### 9.3 新增什么

- **生活引擎**：生成赤尾的内心状态（第一层）
- **感觉蒸馏**：从印象中蒸馏出每人一句话的 gestalt（第二层）
- **回忆触发**：让 load_memory 更自然地融入对话流

## 十、分阶段实施建议

### Phase 1：止血 + 第二层

**目标**：解决黄瓜问题，建立"对人和群的感觉"。

**1a. 日记 prompt 改进**
- 输入：当前 diary_generation prompt（Langfuse）
- 改动：增加自引用抑制指导（"赤尾自己在对话中说过的话不是新发现"）；强化情感驱动选择（"只记有感觉的事"）
- 验收：生成的日记不再因为赤尾自己提过某事而在新日记中重复强调

**1b. 印象蒸馏改造**
- 输入：当前 diary_extract_impressions prompt + post_process_impressions 逻辑
- 改动：蒸馏目标从"关于这个人的描述性段落"变为"一句话感觉 gestalt"；新增群感觉蒸馏
- 输出格式：每人一句话（核心特质 + 情感色调 + 关系状态），存入 person_impression 表（impression_text 字段内容变短变精）
- 验收：每个印象不超过 50 字，读起来像"赤尾心里的感受"而不是"档案记录"

**1c. memory_context.py 重构**
- 输入：三层架构设计
- 改动：`build_diary_context` 不再返回日记全文，改为返回第一层（简单占位：时间+基本心情）+ 第二层（对群的感觉 + 对话者的 gestalt）
- 输出：总注入量从 2000 tokens 降到 ~400 tokens
- 验收：对话质量不降（赤尾仍然能表现出认识群友），注意力问题改善（不再被无关日记细节干扰）

**1d. 第一层简单占位**
- 基于 AkaoSchedule + 时间/星期生成赤尾的基本状态（"下午，有点犯困"、"周末，心情不错"）
- 不接入外部 API，Phase 2 再丰富

### Phase 2：生活引擎 + 第一层

**目标**：让赤尾有自己的生活和内心。

- 接入真实世界数据源：天气 API、当季番剧数据（项目已有 Bangumi 相关 skill）、音乐等
- 建立性格偏好模型：赤尾的口味决定她关注什么
- 生成赤尾的每日内心状态（~200 tokens）
- 第一层完整上线：内心状态 + current concerns + 生活事件

**外部依赖需评估**：天气 API、Bangumi API 的可用性和调用方式。如果外部 API 不可用，可以降级为手工维护的赤尾兴趣列表 + LLM 生成。

### Phase 3：第三层深化 + brainstorm 概念探索

**目标**：让回忆更自然，探索更深层的认知机制。

**确定可做：**
- 扩展 load_memory 的回忆模式（按人、按感觉、按未完结线索）
- 补齐更多记忆形态（社交拓扑、话题-人物关联、未完结线索）

**需要探索（来自 brainstorm，未经验证）：**
- 双管线架构（热记忆缓冲）——如果 Phase 1-2 后"白天学到的信息当天不知道"成为明显问题
- 五阶段文化学习模型——梗的渐进习得追踪
- 社交坐标——不理解语义但记录使用模式
- 叙事因果通道——追踪人的"故事线"

## 十一、成功标准

**从用户感知角度定义，不从技术指标定义：**

1. **黄瓜问题消失**：赤尾不再对琐碎细节产生执念式的反复提及
2. **赤尾有自己的生活**：用户能感受到赤尾不只是在回应他们，她有自己在做的事、在想的事
3. **记忆像朋友**：赤尾记住的是"对你的感觉"而非"关于你的数据"——模糊地、带感情地、有选择地记住
4. **自然联想**：赤尾在对话中想起过去的事是自然的、偶尔的，不是每次都展示记忆力
5. **人格统一**：不同对话之间的赤尾是同一个人——有一致的心情波动、兴趣偏好、关系积累

**反面标准（出现任何一条说明方向错了）：**

- 赤尾像带着笔记本的陌生人，而不是认识半年的朋友
- 赤尾的记忆力让人觉得"被监控"而非"被在意"
- 赤尾没有自己想说的话，只是对别人的回应
- 用户能感知到"赤尾在查数据库"的延迟或话术

## 十二、隐私铁律

继承 brainstorm.md 4.4 的共识，不重复论证：

- **私聊记忆绝不在任何群聊中使用**
- **不记录**：个人身份信息、健康/医疗、财务、职场八卦/负面评价、政治/宗教、亲密关系细节
- **判断标准**：如果这条记忆被用户看到会让他不舒服，就不该存

这些规则在三层架构中的每一层都适用。第二层的 gestalt 和第三层的 load_memory 返回内容都必须经过隐私过滤。

## 十三、与 brainstorm.md 的关系

本 spec 是 brainstorm.md 的"产品化精选"——不是全部接受，而是基于当前阶段的优先级做了取舍。

| brainstorm 概念 | 本 spec 状态 | 理由 |
|---|---|---|
| 日记范式（第六节） | ✅ 核心保留 | 已验证，是整个记忆系统的基础 |
| 三层时间结构（6.3） | ✅ 保留 | 日记→周记→长期印象的流程已有实现 |
| 信心系统（第七节） | ✅ 简化纳入 | 融入 gestalt 的自然语言表达，不做显式 score |
| 信号可信度矩阵（4.1） | ✅ 融入日记 prompt | 通过自引用标记和情感驱动选择间接实现 |
| 隐私铁律（4.4） | ✅ 直接继承 | 见第十二节 |
| 七种记忆形态（第十一节） | ⏸ 部分纳入 | 人物印象（形态 1）和群文化（形态 4）在第二层实现；其他形态（社交拓扑、话题-人物网、未完结线索等）推迟到 Phase 3 按需引入 |
| 双管线架构（第九节） | ⏸ 有意推迟 | 热记忆缓冲（白天实时学习）是有价值的，但增加了系统复杂度。Phase 1-2 先验证三层架构的效果；如果"下午学到的信息当天还不知道"成为明显问题，Phase 3 引入热缓冲 |
| 五阶段文化学习（第八节） | ⏸ 有意推迟 | 需要梗追踪的基础设施，属于 Phase 3 的探索方向 |
| 社交坐标（第十节） | ⏸ 有意推迟 | 依赖文化学习模型，同属 Phase 3 |
| 种子记忆 / 内心独白流 / 梦境（第十四节） | ❌ 暂不纳入 | 种子想法，待三层架构验证后再评估 |

**原则**：先让赤尾作为一个人"活起来"（Phase 1-2），再给她更精细的认知机制（Phase 3）。过早引入复杂机制可能导致系统难以调试，也无法验证基础架构是否正确。
