# Life Engine 重设计：Agent Team 架构

## 问题

赤尾的日程连续 7 天下午"带老相机去旧巷子拍叶樱"，晚上"看新番/整理桌面"。生活极度单调。

### 根因链

1. **三层规划漏斗**：月计划定调"叶樱+胶片+抹茶" → 周计划继承 → 日计划只能在极窄的活动池里选择
2. **persona_core 的摄影引力**：摄影在人设中描写最生动（"胶片那种不可撤回的感觉很像人的记忆"），是唯一能写成"下午出门做事"的兴趣，LLM 自然偏向它
3. **Ideation 搜索太弱**：搜出来的泛素材（"花粉高"、"city walk 很火"）竞争不过 persona 的摄影引力
4. **tick 层面的重复是表象**：真正无聊的是日程内容本身

### 实验验证过程

共进行 8 轮实验，关键发现：

| 实验 | 发现 |
|------|------|
| 外部刺激强度对比 | 手工构造的高质量具体素材能打破摄影引力 |
| 姐妹小剧场 | 家庭互动直接占满注意力，完全不去拍照，情感层次远超之前 |
| 真实搜索策略（4种） | 搜索引擎天花板：返回内容农场和旅游攻略，搜不到生活碎片 |
| 大范围搜索（12方向） | 全线平庸（最高3-4/5），确认通用搜索不是获取生活素材的正确工具 |
| 网状搜索+小剧场 | 20条具体搜索词+小剧场=最高质量，但搜索词硬编码不可持续 |
| 策展人模式 | 第三方LLM生成搜索词质量好，但连续5天后搜索词本身趋同（小河直街5/5天） |
| 策展人漂移测试 | "天天拍照"变成"天天嘴硬吃春笋"——模式被替换不是被消除 |
| **Agent Team** | **发散和收敛分离，质量最高：情感深度、多样性、全天弧线均远超其他方案** |

## 方案：Agent Team 架构

### 架构总览

```
┌─ Wild Agent A（互联网漫游）──┐
│─ Wild Agent B（城市观察）    ──┤
│─ Wild Agent C（兔子洞）      ──┼→ 主 Agent（persona 视角筛选）→ Writer → daily schedule
│─ Wild Agent D（情绪天气）    ──┤
│─ 真实搜索锚点（3-5条）      ──┘
│
└─ 姐妹小剧场 Agent ──────────────────→ Writer（三姐妹共享消费）
```

### 砍掉的部分

- **月计划**：删除。它是窄化漏斗的顶端，定调了整月只做"摄影+追番+抹茶"
- **周计划**：删除。同上
- **Ideation Agent**：用 Wild Agent Team 替代
- **Critic Agent**：暂时保留，但输入从"跟前3天对比"改为对 agent team 产出的日程做质量检查

### 组件详细设计

#### 1. Wild Agents（4个，并行运行，不知道 persona）

每个 wild agent 从不同维度想象"一个 18 岁女生的世界里今天飘过什么"。**它们不知道 persona 是谁**——不知道她叫赤尾、不知道她喜欢摄影、不知道她住杭州。它们只知道"18 岁中国女生"这个最基本的画像。

| Agent | 职责 | 输入 | 输出 |
|-------|------|------|------|
| A: 互联网漫游 | 想象她刷手机看到什么 | 日期、季节 | 8-10条具体的 app 推送内容（B站视频标题、小红书帖子、微博话题等） |
| B: 城市观察 | 想象她出门遇到什么细节 | 日期、季节、天气 | 8-10条路上的画面（声音、气味、光线、路人、街角变化） |
| C: 兔子洞 | 生成让人停不下来的冷知识 | 无 | 8-10条具体的冷知识/发现/反直觉事实 |
| D: 情绪天气 | 想象这个季节的集体感受 | 日期、季节 | 6-8条身体感受、普遍情绪、社交媒体 vibe |

**关键设计决策**：Wild agents 不带地域限定（不写"杭州"），避免搜索词地理趋同。城市观察 agent 只写"南方老城区"级别的描述，Writer 自己会锚定到杭州。

#### 2. 真实搜索锚点（3-5条）

仅搜索事实性信息，用于锚定现实：

- 今天的天气
- persona 正在追的番的更新状态
- 本地 1-2 件真实事件（新开/关店、展览等）

搜索词由系统构造（不由 LLM 生成），基于：
- persona 的地理位置（配置项）
- persona 当前追踪的媒体列表（从 life state 或专用字段读取）

#### 3. 主 Agent（persona 视角筛选）

输入：4 个 wild agents 的全部产出 + 真实搜索锚点（约 40 条素材）

任务：用 persona 的视角从中"捡"她会在意的 6-8 条。

Prompt 要点：
- 注入 persona_lite（不是完整 persona_core，避免过度引导）
- 明确要求多样性："不要全是同一类"
- 明确要求包含"她不会主动搜但刷到会记住的"
- 输出时标注"为什么她会在意这个"

#### 4. 姐妹小剧场 Agent

独立于上述流程，为三个 persona 生成共享的家庭事件。

输入：
- 三姐妹 + 主人的**性格轮廓**（不含兴趣细节，只有性格特质和关系）
- 日期、星期几
- 前一天的小剧场摘要（避免连日重复）

输出：5-6 件家庭琐事

**三个 persona 共享同一份小剧场**——赤尾的日程里写"绫奈回来书包一丢"，绫奈的日程里也会写同一件事但从她的视角。

#### 5. Writer

输入：
- persona_core（完整人设）
- 主 agent 筛选后的 6-8 条素材
- 姐妹小剧场
- 昨天的日程摘要
- 日期/星期

输出：日程手帐（上午/下午/晚上，脑内活动为核心）

### 数据流

```
每日凌晨触发（或按需触发）：

1. [并行] Wild Agent A/B/C/D 生成素材           ~4次 LLM 调用
2. [并行] 真实搜索 3-5 条                       ~1次搜索 API
3. [并行] 姐妹小剧场生成                        ~1次 LLM 调用
4. [串行] 主 Agent 筛选（输入: 1+2 的产出）      ~1次 LLM 调用
5. [串行] 对每个 persona 生成日程                ~3次 LLM 调用（akao/chinagi/ayana 各一次）
6. [可选] Critic 检查质量                        ~3次 LLM 调用

总计：~12-15 次 LLM 调用/天
```

### 与现有系统的关系

#### 保留不变

- **life_engine_state 表**：仍然 append-only，tick 仍然每分钟检查
- **life_engine_tick prompt**：不变，它消费 daily schedule 的内容
- **persona_core / persona_lite**：不变
- **bot_persona 表**：不变

#### 修改

- **akao_schedule 表**：仍用于存储 daily plan，但不再存月/周计划
- **schedule.py**：
  - 删除 `generate_monthly_plan()`、`generate_weekly_plan()`
  - 重写 `generate_daily_plan()` 为 agent team 架构
  - 删除 `_run_ideation()`（被 wild agents 替代）
  - 保留 `_run_writer()` 和 `_run_critic()`（prompt 需更新）
- **cron.py**：删除月/周计划的 cron 触发

#### 新增

- **wild_agents.py**：4 个 wild agent 的 prompt 和调用逻辑
- **sister_theater.py**：姐妹小剧场生成逻辑
- **Langfuse prompts**：
  - `wild_agent_internet`、`wild_agent_city`、`wild_agent_rabbithole`、`wild_agent_mood`
  - `sister_theater`
  - `daily_curator`（主 agent 筛选）
  - 更新 `schedule_daily_writer`

### 连续性追踪（后续迭代）

本次不实现，但预留方向：

- **追踪"她正在做的事"**：追到第几集、在看什么书、说过想去哪家店
- **追踪"她做过的事"**：前 N 天的活动摘要，供 wild agents / critic 参考以避免重复
- **persona 兴趣演化**：从日程和对话中提取新兴趣，缓慢更新 persona

### 成本估算

现有方案：月计划 1 次 + 周计划 1 次 + 日计划 3 次（ideation+writer+critic）× 3 persona = ~11 次/天

新方案：wild agents 4 次 + 搜索 1 次 + 小剧场 1 次 + 主 agent 1 次 + writer 3 次 + critic 3 次 = ~13 次/天

成本基本持平，但 wild agents 用 offline-model（gpt-5.4），可以考虑降级到更便宜的模型（wild agents 不需要太强的推理能力，需要的是想象力和多样性）。

### 风险与缓解

| 风险 | 缓解 |
|------|------|
| Wild agents 自身也可能趋同（每天生成类似内容） | 1. 不给它们 persona 信息，减少锚定 2. temperature 拉高 3. 每次运行随机打乱 agent prompt 中的示例 |
| 主 agent 筛选时仍偏向摄影 | 用 persona_lite 而非完整 persona_core；prompt 明确要求多样性 |
| 姐妹小剧场连日重复 | 输入前一天的小剧场摘要，prompt 要求避开 |
| 日程缺乏连续性（昨天说想去的店今天忘了） | 后续迭代加入"ongoing threads"追踪 |
