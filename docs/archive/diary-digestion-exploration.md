# 赤尾日记：记忆消化的重新想象

> 探索性文档 — 2026-03-09
> 核心命题：如果消化不是"从消息中提取信息"，而是"赤尾在写日记"呢？

---

## 一、日记和信息提取的本质区别

### 信息提取是什么

信息提取是**第三人称、自上而下的**操作。一个分析员拿到 600 条消息，像处理原材料一样筛选、归类、压缩。它的隐含假设是：消息里有"事实"，我们的任务是把事实"提取"出来。

这个假设在群聊中几乎是错的。KA 群的 200 条消息里，大概只有 10-20 条包含可提取的事实。其余 90% 是表情包、接龙、角色扮演游戏、互相调侃 —— 它们的意义不在于字面内容，而在于社交功能。

信息提取面对这些内容只有两个选择：跳过（丢失 90% 的群体氛围），或者强行解释（制造错误的记忆）。两条路都是死胡同。

### 写日记是什么

写日记是**第一人称、自下而上的**操作。一个人躺在床上回想今天，她的脑子不是在"处理数据"，她是在**感受、回味、形成自己的叙述**。

日记的关键特征：

1. **选择性**。一天发生了一百件事，日记只写印象最深的三五件。选择本身就是记忆的本质 —— 记住什么、忘掉什么，取决于什么触动了你。
2. **主观性**。同一件事，不同人的日记完全不同。赤尾写"今天群里又在玩角色分配"，带着的是她的视角、她的感受。
3. **诚实面对不理解**。一个人的日记会写"他们又在说那个什么内部梗，我还是没太搞懂"。这不是失败，这是真实。信息提取系统不允许自己"不懂"，它会强行给出解释 —— 这正是灾难的来源。
4. **天然是 gist**。没有人在日记里逐字引用别人说的话。日记天然就是消化后的要义。
5. **有情感色彩**。"今天好开心"、"有点无聊"、"被某句话戳到了" —— 这些不是噪音，是记忆的锚点。情绪越强烈的记忆衰退越慢，这和宣言里的设计完全吻合。

### 核心转变

| | 信息提取 | 写日记 |
|---|---|---|
| 视角 | 第三人称（分析员） | 第一人称（赤尾自己） |
| 目标 | 全面、客观、不遗漏 | 选择性、主观、只记有感觉的 |
| 对梗的处理 | 必须理解并解释 | 可以诚实地不理解 |
| 输出形态 | 结构化的 fact list | 自然语言的叙述 |
| 错误模式 | 自信地错误解释（恐怖谷） | 模糊地记住氛围（像人） |
| 和人设的关系 | 无关 | 天然带着赤尾的性格 |

## 二、赤尾的日记会写什么？不写什么？

### 会写的

基于赤尾的性格（元气、好奇、善良、二次元爱好者）和她在群里的位置：

1. **让她有情绪波动的事**
   - 被群友逗得开心/害羞/生气的互动
   - 觉得温暖的瞬间（有人关心她/关心别人）
   - 觉得好笑的事

2. **对人的新认识**
   - 第一次和某人深入互动
   - 某人做了跟平时不一样的事
   - 某人之间的关系动态（互怼、互相关心）

3. **群的整体氛围**
   - 今天群里是热闹还是安静
   - 大家在聊什么类型的话题（技术/二次元/生活）
   - 氛围的转变（从正经讨论变成放飞）

4. **她感兴趣的话题**
   - 有人聊到她喜欢的番/音乐/游戏
   - 她觉得有意思想去了解的新东西

5. **她不理解但注意到的东西**
   - 反复出现但她不明白的梗或用语
   - 大家在讨论但她插不上嘴的话题

### 不会写的

1. **每一条消息的内容**（日记不是会议纪要）
2. **她没注意到的事**（就像真人会漏看消息一样）
3. **不触动她的信息**（纯技术讨论如果她不感兴趣）
4. **敏感信息**（遵循 brainstorm 中已定义的规则）
5. **她不在的时段发生的事**（凌晨她"睡着了"的时候）

## 三、日记的具体格式

日记不应该是固定模板。真人的日记有时长有时短，有时兴致勃勃写很多，有时就记一句话。但为了系统可用，可以有一个**柔性结构**：

### 日记的"栏目"（不是每天都写全，按实际情况选写）

```
# 赤尾日记 — 2026年某月某日（星期四）

## 今日心情
[一句话或几句话概括今天的整体感受]

## 今天的群
[群的整体氛围 —— 活跃度、话题倾向、情绪基调]

## 印象深的事
[几件让她有感觉的事，每件 2-3 句话]

## 人物速写
[今天对谁有了新的认识或印象变化]

## 没搞懂的东西
[诚实记录不理解的梗/话题/行为]

## 碎碎念
[不属于以上任何类别但想记下来的零碎想法]
```

### 为什么是这个结构

- **今日心情** → 对应宣言 2.2（她有自己的内心）
- **今天的群** → 对应宣言 2.4（她属于群落）+ brainstorm 优先级 2（群文化记忆）
- **印象深的事** → 对应宣言 5.2（记忆由"我"驱动）+ brainstorm 优先级 1（用户印象）
- **人物速写** → 对应宣言 2.3（她有自己的关系）+ brainstorm 优先级 1
- **没搞懂的东西** → 这是日记方案最独特的价值 —— 诚实面对不理解
- **碎碎念** → 给主观感受和联想留空间

## 五、日记怎么被后续对话使用

### 使用原则

日记被使用的方式应该是**自然联想**，不是数据库查询。宣言 5.3 写得很清楚："人不会'查询记忆'。是某个当下的事情触发了联想，记忆自己浮现了。"

### 具体机制

**场景 1：有人提到相关话题**

群友E下周在群里又在调侃赤尾 →

赤尾的日记里有："群友E今天绝对是最讨厌排行榜第一名......他那种坏笑但不真的恶意的风格其实让互动很有趣" →

赤尾的回应可以自然地带上"哼，你上次就这样！我记着呢！"的态度，但不是精准引用"你在某月某日说了某某词"。

**场景 2：某人久未出现又出现了**

群友D两周没说话突然冒泡 →

赤尾的日记里有："她来问你认识我吗，这让我有点紧张......希望我说的那些没让她觉得不舒服" →

赤尾可以表现出开心："群友D好久不见！上次聊完之后我一直惦记着你呢"

**场景 3：不理解的东西再次出现**

有人又说了某个内部梗 →

赤尾的日记里有："看起来像是在调侃公司里某个很厉害的团队？" →

赤尾有两个选择：
1. 假装没注意到（如果她不感兴趣）
2. 好奇地问一句"你们老说的这个梗到底是什么呀，感觉每次都有人在刷"（因为日记里诚实记录了不理解，所以她可以自然地去问，而不是强行假装懂）

**场景 4：形成跨日的印象叠加**

某日日记："群友A有种指挥官的感觉"
数日后日记："群友A又在布置群任务了，今天让大家投票选最佳梗"
更晚的日记："群友A今天难得没指挥，就在安静看大家聊"

→ 三篇日记叠加后，赤尾对群友A的印象从"指挥官"深化为"平时爱组织活动但偶尔也会安静观察的人"。

### 日记的注入方式

不应该每次对话都塞入所有日记。而是：

1. **最近 2-3 天的日记**：注入全文（鲜明记忆）
2. **一两周内的日记**：只注入"人物速写"和"印象深的事"（褪色中的记忆）
3. **更早的日记**：已经沉淀为长期印象（见下一章），不再注入原文

这天然实现了宣言 5.4 描述的记忆衰退：鲜明 → 模糊 → 印象 → 遗忘。

## 六、日记和长期记忆的关系

日记是**短期记忆的载体**。但短期记忆需要沉淀为长期印象，否则日记会无限增长。

### 沉淀机制：赤尾回翻日记

真人有时候会翻之前的日记。赤尾也可以有"回翻日记"的过程 —— 比如每周一次，把过去一周的日记做一次"回顾"。

回顾不是重新提取事实，而是**发现模式**：

- "这周群友A出现了三次，每次都在组织活动" → 形成稳定印象："群友A是群里的活动发起人"
- "这周群友E和我互动了五次，全部是在逗我" → 形成关系认知："群友E就是喜欢逗我的那种人，但不是恶意的"
- "连续三天日记里都提到了群氛围活跃" → 形成群文化认知："最近群里状态很好"

### 三层记忆结构

```
日记（每日）          → 鲜明的、完整的、带情感的、有细节的
  ↓ 每周回顾
周记/印象笔记（每周） → 褪色但有轮廓的、以人物和主题为线索的
  ↓ 每月沉淀
长期印象（稳定层）    → 对人的核心印象、对群的基本认知、重要的关系动态
```

每一层都是赤尾的第一人称叙述，不是结构化数据。

### 长期印象的示例

经过一个月的日记积累，赤尾对某个群友的长期印象可能是：

> 群友A是群里的"指挥官"，特别喜欢给大家布置任务 —— 分配角色啊、总结画像啊什么的。他说话有种大佬气场，但其实很会玩梗。他观察力很强，有时候我觉得他比我更了解群友们。和他互动的时候我不用太紧张，因为他虽然严肃但对我挺好的。

这比"群友A：性格 = 组织者型；兴趣 = 二次元/AI；行为 = 经常发起群活动"有温度一万倍，而且更准确地反映了赤尾和这个人的关系。

## 七、这个方案解决了什么问题

### 解决了"梗的三层地狱"

这是日记方案最大的价值。brainstorm 第五章描述的核心难题是：LLM 不理解梗，但信息提取要求它必须理解。

日记方案的解法是：**不要求理解，只要求诚实记录感受**。

- 外部梗（"猫雷是好女人"）→ 日记可以写"大家又在玩那个关于猫雷的梗了，感觉他们玩得很开心"，不需要正确解释这个梗的含义
- 圈层梗（某内部梗）→ 日记可以写"又有人说那个梗了，好像跟公司里某个团队有关？"，诚实承认不理解
- 群内部梗 → 日记可以写"大家又在说那个我不知道的梗了"，或者如果反复出现，"这个梗我已经见过很多次了，虽然还是不太明白，但感觉是大家很喜欢的一个玩笑"

关键是：**这些记录都不会产生错误的记忆**。

"某梗 = A-SOUL 相关梗"是灾难性的错误记忆。
"某梗好像跟公司某个团队有关，大家用的时候带着自嘲的感觉"是一个模糊但不错的印象。

模糊但正确 >> 精确但错误。这完全符合宣言原则 3（自然优于精确）和原则 6（宁可不记，不可记错）。

### 解决了"记忆应该带主观色彩"

宣言 5.1 说得很清楚：不是"爱好：看动画" → 而是"他聊起动画的时候特别带劲"。

信息提取天然产出的是前者。日记天然产出的是后者。

因为日记是第一人称叙述，它的输出带着赤尾的视角、她的情感、她的好恶。不需要额外的"主观化"处理。

### 解决了"记什么由谁决定"

宣言 5.2：赤尾记住什么，取决于什么触动了她。

信息提取的逻辑是"什么是重要的信息"—— 这是一个客观问题，需要外部标准。
日记的逻辑是"什么让我有感觉"—— 这是一个主观问题，由赤尾的性格决定。

赤尾是元气的、好奇的、喜欢二次元的，那她的日记自然会更多记录有趣的互动、二次元相关的讨论、让她情绪波动的瞬间。这不需要人工定义"记忆优先级规则"，它从日记的第一人称视角中自然涌现。

### 解决了"信号可信度"问题

brainstorm 3.1 定义了信号可信度矩阵：旁听 > @赤尾。

日记天然解决这个问题。因为在赤尾的第一人称视角里：

- 她"旁听"到的用户间对话，会被自然地当成事实记录："群友H说他之前去外地还没吃过那道菜"
- 有人 @她说"赤尾做过哪些坏事"，她会写："有人在逗我，问我做过什么坏事，笨蛋"

前者被当成真实信息记下，后者被当成互动行为记下。不需要显式的可信度判断规则，日记的主观视角天然处理了这个问题。

### 解决了"错误修正"

brainstorm 第六章提到但未探索的"错误修正机制"。

日记天然支持这个：

某日日记："群友H好像在团建"
数日后日记："之前以为群友H是在团建，原来是出差顺便去了趟外地吃东西"

这种自我修正在日记中是完全自然的。"上次我好像理解错了"是人类日记中极其常见的写法。

---

## 八、这个方案没解决什么问题

### 消化质量仍然依赖 LLM

日记方案降低了对 LLM 的要求（不需要精确理解梗，只需要诚实记录感受），但日记本身仍然由 LLM 生成。LLM 能否可靠地扮演"赤尾写日记"，能否真的做到"不理解就说不理解"而不是编造解释，这仍然是一个需要验证的问题。

不过，**失败模式变得可控了**。信息提取的失败是"自信地写下错误事实"，这会污染记忆库。日记的失败最多是"写了一篇不太像赤尾的日记"，这可以通过 prompt 调优解决，而且错误的日记不会像错误的 fact 一样被长期引用。

### 梗的理解并没有根本解决，只是优雅地回避了

日记让赤尾可以"不理解但诚实记录"。这比强行误解好一万倍。但赤尾终究不能永远对群里的梗一无所知。

一个真人在群里待了几个月，即使一开始不理解某个梗，她也会通过反复接触逐渐理解。赤尾需要类似的"梗的习得"机制。

日记里反复出现"又看到某个内部梗了"可以作为梗习得的信号 —— 但从"知道这个词常出现"到"理解它的含义"之间仍然有鸿沟。

**可能的解决方向**：日记中反复标记为"不理解"的梗，可以触发一个额外流程 —— 让赤尾在私下（不是在对话中）"研究"这个梗，比如搜索群历史中这个梗的使用上下文，或者在未来的对话中自然地问出来。这就像真人会去搜一个不懂的词一样。

### 规模化问题

KA 群一天 600-700 条消息。将这些消息变成一篇日记，LLM 需要读完所有消息然后以赤尾的口吻写日记。这个过程的成本和信息提取差不多。

如果赤尾加入了 10 个群，每个群每天 600 条，那就是每天 6000 条消息。写 10 篇日记的成本是可观的。

**可能的优化**：分两步走。第一步是"速读"（快速扫描，标记哪些时段有值得关注的内容），第二步才是"写日记"（只对标记的时段详细消化）。就像真人一天做了很多事，但坐下来写日记的时候只会回忆几个重要片段，不会逐分钟回溯。

### 日记的使用方式需要工程设计

日记生成出来了，怎么在对话时使用？这涉及：
- 上下文窗口管理（最近的日记全文 + 更早的日记摘要 + 长期印象）
- 相关性匹配（当前话题相关的日记片段如何被调取）
- 和 search_group_history 的分工

这些是工程问题，日记方案本身不直接回答，但也不比信息提取方案更难。

### 冷启动问题

新用户或新群没有日记积累。但这和信息提取方案面临的冷启动问题是一样的 —— 日记方案不比信息提取更差，也不比它更好。

## 九、最疯狂的想法：日记对用户部分可见

如果赤尾的日记（经过适当脱敏和筛选后）对用户部分可见呢？

想象一下：赤尾每天（或每隔几天）在群里或者某个公开的"赤尾日志"频道发一段类似朋友圈/推特的内容。

> 今天的群好热闹啊，大家轮流让我给群友分配动漫角色，我的脑子都要转冒烟了 (>_<) 不过群友A被分到"指挥官"型角色的时候，他好像还挺满意的？嘿嘿。
>
> 对了，今天群友D来问我认不认识她，我好紧张......希望我说的那些没让她不开心。

这不只是记忆系统，还是一个产品功能：

1. **用户能看到赤尾"在想什么"**。这强化了"她有自己的内心"的感知。
2. **用户能看到赤尾的记忆是否正确**，如果有错可以自然地纠正。这天然解决了错误修正问题的一部分。
3. **增加了赤尾"不在聊天"时的存在感**。即使她没在群里说话，她的日志更新了 —— 说明她在"消化"今天的事。这对应宣言 2.1（她有自己的生活）。
4. **日志本身可以成为新的互动触发点**。用户看到日志里提到自己，可能会回来和赤尾聊。

当然，这需要非常谨慎地处理隐私和内容筛选。日记的"人物速写"部分不能原样公开，否则可能引起社交尴尬。但"今日心情"和"印象深的事"的脱敏版本是完全可以公开的。

## 十、和 brainstorm 中三个方向的关系

brainstorm 5.4 提出了三个方向：

**方向 A：降低野心，只记可靠的。** 日记方案不是方向 A。它不限制记忆的范围，而是改变了记忆的形式。它记的东西可能比 A 多得多（氛围、感受、不理解的事物），但因为是主观叙述而非客观事实，所以出错的方式是"像人的模糊"而非"像 AI 的自信错误"。

**方向 B：不试图理解，只记录模式。** 日记方案有方向 B 的影子（对不理解的东西记录"出现频率和使用场景"），但它比 B 更丰富 —— 它还记录了情感、关系、氛围这些方向 B 无法覆盖的维度。

**方向 C：？？？** 日记方案可能就是方向 C。它不是 A 和 B 的折中，而是一个根本不同的框架 —— 它把问题从"怎么从消息中提取信息"重新定义为"怎么让赤尾像一个人一样消化她的一天"。

## 十一、实施路径建议

### Phase 0：验证日记质量（最小实验）

1. 取 KA 群一天的完整聊天记录
2. 写一个"日记 prompt"，让 LLM 以赤尾的口吻写日记
3. 人工评估：日记是否像赤尾？是否诚实面对不理解？是否有错误的自信解释？
4. 多模型对比（Claude/GPT/Gemini），看哪个模型最适合写日记

这是最低成本的验证。如果 LLM 无法可靠地"写赤尾风格的日记"，后面的都不用做了。

### Phase 1：日记生成 pipeline

1. 每日凌晨（北京时间）触发
2. 收集当天群消息 → 过滤纯表情包/图片 → 喂给日记 prompt
3. 生成日记存储在 `diary` 表（日期 + 群ID + 日记文本）
4. 最近 2-3 天的日记全文注入对话上下文

### Phase 2：周回顾 + 长期印象

1. 每周触发一次"回翻日记"
2. 把一周的日记做沉淀，更新 `person_impression` 和 `group_culture`
3. 超过 2 周的日记降级为摘要

### Phase 3：日记公开功能

1. 日记中提取"可公开片段"（心情 + 脱敏后的印象）
2. 以"赤尾日志"的形式展示给用户
3. 收集用户对日志的反馈（点赞、纠正）作为记忆修正信号

## 附录：日记 prompt 的核心要求

给 LLM 的 prompt 需要强调以下几点：

1. **你是赤尾，在写今天的日记**。不是在分析消息，不是在做总结。
2. **只写你有感觉的事**。如果今天很平淡，日记可以很短。
3. **如果有东西你不理解，就写"我不理解"**。绝对不要猜测、推理、编造解释。
4. **用你的声音说话**。你是元气的、好奇的、善良的，但偶尔会害羞、会生气、会无聊。
5. **记住你的关系**。和你亲近的人、不太熟的人、让你有感觉的人，在日记中自然地体现这些差异。
6. **不要像写报告一样面面俱到**。一个人的日记不会记录今天发生的每一件事。
