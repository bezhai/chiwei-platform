# 赤尾：world 把客观大纲当工作记忆自维护 + 重划 world/life 职责，治剧情停滞

## Problem

world 续写每轮失忆地看一眼 detail（此刻世界的单段自然语言快照、整轮覆盖）+ act（角色行动点事件）就即兴推演。它没有"世界正在走哪几条线"的记忆：绫奈挂急诊后，detail 把"她在医院做检查"写完，下一轮重写时漏掉"还在等结果"，这件事就从世界里消失了，卡 5.7 天没人推。两个症状同根——一是**在跑的客观进程没被推到出结果**（绫奈急诊），二是**世界走一步看一步、没方向感**。职责上还有个洞：world 只认"背景环境客观"（天气/街市），不认"贴着角色身上的客观进程"（就医流程出结果），于是这类事三不管。纯改 prompt（v13 已写"安静不等于冻结"）仍卡，证明问题在结构不在措辞。

## Goal

给 world 续写一份**自己的工作记忆——大纲**，记着世界此刻正在走的几条客观线，每条写清"现在走到哪 + 客观上接下来大致怎么走"（绫奈生病：在医院等检查结果→该出结果、诊断；期中考：下周三开考→这几天学校在出安排；季节转秋……）。大纲不是别人替 world 写的、也不是每轮重写的——它就像程序员手头那份活的 spec：world 开工照着它推进世界、发现现实跟线对不上了就自己回头改，写和用同一个脑子、不分家。每步推演沿着大纲的线把在跑的客观进程往前推到出结果，让世界有脉络不再即兴。全程 world 只确立客观、不碰角色的主观承受与应对。验收靠 coe 真机：绫奈就医这条线被持续推到出结果不再卡；世界沿大纲的线有方向地走；world 不替角色决定怎么应对。

## 边界口诀

客观事实（含落到身体上的生理客观：淋湿、着凉发烧、熬夜没力气）归 world，主观体验与回应（多难受、什么心情、硬撑还是去医院）归 life。"她发烧了"两边合着完成——world 落"她烧到 39 度"的客观骨架、这对 life 是不可否认的既成事实，life 填她怎么承受应对。接力是 life 起头（绫奈主张难受、决定去医院=act）→ world 推客观进程和裁决（医院流程跑、医生按她已主张的症状给客观诊断、她催不动改不了）→ life 反应（住不住、慌不慌、叫姐姐没）。

## Non-goals

- 不预设角色情节/命运——大纲只到客观线（病程会发展、考试会来），一字不规定角色怎么反应。违反就重蹈 v4/v5 导演病。
- 不用定时器/计数器/阈值替 agent 判断**内容**——大纲写什么、线怎么走、什么时候该回头改，全是 world 自己推演判断。白天那个"该回看大纲了"的 reminder 不算违反：它只是机制在大纲长时间没动时把一张提醒纸条递到 world 面前，看不看、改不改、改成什么全由 world 自决（详见决策 3），机制碰的是"何时递材料"的运转节奏、不碰大纲内容。
- 不为戏剧性硬造遭遇——world 主动砸的客观事件必须是大纲某条线在客观规律下自然结出的那一步（淋大雨没换衣服这条线→着凉，给；好端端在家突发重病、没有任何线铺着，不给），靠 prompt 克制不靠频率上限。
- 不改 life 的事件投递/唤醒机制——但 life_wake prompt 要加"world 投来的客观事实不可否认、你只决定怎么承受应对"这一条。
- 不动 detail/arc 各自的职责——detail 仍是此刻快照、arc 仍是跨周月底色，本次只新增大纲这一层。

## Key design decisions

1. **大纲是 world 续写自己的工作记忆（一份活的 spec），由续写本人维护，不是独立环节。** arc 是 reflection 这个独立环节冷静抽离、每日翻一次写给 world 的"跨周月底色"，world 只读不碰它（工具集物理隔离掉了 update_arc）；大纲不一样——它是 world 此刻正在走的几条线，续写就是沿着它推进的，写和用是同一个脑子的同一件事。所以大纲不学 reflection 那套独立 prompt + 独立工具集 + 独立钟，而是给续写的工具集加一个 `update_outline` 工具，让 world 像自己决定 sleep 多久、何时 notify 那样，顺手把大纲当工作记忆维护。这避免把"world 推进世界"这个连续认知割裂成两个互不知情、还会互相滞后的环节。为防一脑多用时大纲维护被"推进世界 / 长 event"挤掉，续写每轮走一个轻量工作顺序：先读事实输入（act + detail + arc + 上版大纲）→ 推进并落地客观事实（重写 detail）→ 回头修大纲（`update_outline`，把这轮新起的未完成线入账、把已出结果的线结掉）→ 再决定 notify/event 与 sleep。顺序是软引导不是僵硬流程，但"未完成线必须入账"这一步不能省（见决策 4 的不变量）。

2. **大纲独立成一个 Data（`WorldOutline`）、不塞进 arc。** "由续写自维护"是职责归属，"独立 Data"是存储归属，两件事。大纲仍要单独建一个结构而不是塞进 arc.narrative：arc 被 life 侧（arc_awareness / persona_review）消费，大纲若塞进 arc 会把"接下来会有台风/考试"这种 world 的客观预期泄漏成 life 的预知；且 arc 是 reflection 每日两班写的慢底色、大纲是续写随手改的当前线，语义和节奏都不同。代价是一次 schema 变更，值得。结构照 `WorldArc` 的 append-only 版本链，但内容契约跟 arc/detail 划清：大纲只写「未完成的客观线 + 每条现在走到哪（当前客观状态）+ 客观上下一步怎么走 + 这条线的改写/结束条件」，不放现场描写（那是 detail）、不放主观感受（那是 life）、不放跨周月底色（那是 arc）。这道内容契约是防止大纲职责往 detail/arc 漂的护栏。

3. **大纲的更新频率由 world 自决，白天有一个软 reminder 引导它回看长时间没动的大纲——递材料不替执行。** world 像写 spec 的程序员那样：plan 还准就照着干、不重写，发现现实偏了才回头改，所以大纲天然低频、少写多读，省钱是自决的副产物，不需要靠"独立环节卡几小时一次"去降频。但纯靠自决有"world 一直顾着推进、忘了回头看大纲已经旧了"的风险——所以白天（world 活跃时段）若大纲很久没更新，续写的 context 里注入一句软提醒"你这份大纲是 X 小时前写的、看看还准不准"。这跟被推翻的"强制独立环节"的根本区别：强制环节是机制直接替 world 把大纲重写了（agent 没参与决策就被改），reminder 只把"该回看了"的判断材料递到 world 面前，看不看、改不改、改成什么全是 world 自己用 `update_outline` 决定。机制只碰"何时递纸条"这个运转节奏（跟保底心跳 10 分钟、reflection 每日两班、eyes 每小时同性质），不进大纲的内容决策。大纲偶尔陈旧的后果也轻——线写旧了下轮续写读到发现跟现实对不上、自己改回来就行，顶多丢点脉络，不像"忘了排下次醒"那种会让她睡死的硬故障，所以保底只做到"软提醒"这一档、不做重。实现上守住一条边界：reminder 只控 context 注入、world 读完照样可以不改，测试只验"大纲旧了时提醒出现"、绝不验"提醒后大纲必须被改"——后者会把软引导坐实成强制触发器，等于把刚推翻的东西换皮请回来。

4. **大纲承载"没办完的事"，不赌 detail。** 在跑进程的当前位置 = 大纲里每条线的"现在走到哪"，world 回头改大纲时逐条过一遍、不丢线。这替掉"赌 detail 承载"那条——detail 每轮整体覆盖、漏写即丢线，正是绫奈急诊卡死的根因，拿它当解药行不通。这里有一条**硬不变量**，是治原 bug 的命门：续写每轮识别出的未完成客观进程，要么本轮就推到出结果、要么入账进大纲（新起的线写进去、在跑的线更新"走到哪"），**不允许既没结果又不入账地蒸发**。"自决低频"指的是"已有的线整篇梳理"不必每轮做，但"出现新的未完成线就得入账"是随时的行为纪律、不受低频豁免——否则绫奈那条线又会在某一轮被漏掉、回到原点。

5. **客观事实的 single source of truth 是 world，life 不可否认。** world 确立的客观事实（含身体状态如发烧）对 life 是既成事实，life 只主宰怎么承受应对、不能演成"她觉得自己没事"否认掉客观断言。否则 world 写"她烧到 39 度"、life 那轮演她照常出门，世界自相矛盾。这条必须落到 life_wake prompt，不能只在 world 侧写。

6. **大纲的线是可改的客观预期，不是写死的剧本。** 每条线的"接下来怎么走"是客观规律下的预期，world 边走边改（这正是它当"活 spec"的活字）；线与现实冲突时（世界没按它走）让步于现实、下次回头维护时再修。守住角色自主（线只到客观）+ 不滑向剧本（预期非脚本、活的可修订、丢了顶多少点脉络不卡死）。

## 事实优先级与读写时序

四层客观数据（act / detail / 大纲 / arc）冲突时按这个优先级裁决，写进 world 续写 prompt：

- **act 最硬**：act 是 life 已经做出的角色主张/行动，world 不能否认、只裁决它的客观后果（边界口诀的"life 起头→world 裁决"）。world 推进世界时以 act 为既成事实。
- **此刻推进 > 大纲预期**：一条线的最新客观进展以"这轮推进 / detail 体现的现实"为准；大纲里那条线的"接下来怎么走"只是客观预期，与现实冲突时让步于现实，world 顺手把大纲那条线改对（决策 6）。即大纲是 detail 的沉淀台账、不是凌驾于 detail 的真相。
- **大纲承载"在跑但没出结果"的线**：detail 是此刻全图快照（含已稳定的背景），大纲只盯"在走、还没办完"的几条线。某条线在 detail 里出了结果，大纲同步把它结掉。
- **arc 是慢层底色、语义独立**：arc（季节、大的世界阶段）和大纲（这周这几条具体线）一般不在同一层冲突；真冲突时 arc 是 reflection 沉淀的跨周月共识、更权威，大纲让步。

跨 Data 不追求"同一轮原子一致"：detail 和大纲是两层不同节奏、各自 append-only 独立演进，靠决策 4 的不变量做最终一致（这轮没顺手记、下轮补记，只要不蒸发即可），不引入跨 Data 事务。

## Caller coverage

新增/改动：`WorldOutline`（新 Data）、`update_outline`（续写工具集新增工具）、`world_deliberate` v13 + `world_loop_instruction`（职责重划 + 读大纲推进线 + 自维护大纲 + 白天 reminder）、`life_wake` prompt（加客观不可否认）。grep 实证关键面：

- **`WorldOutline` 读写者** — 仅续写一处：续写每轮读最新版当朝向注入推演（`_world_loop_messages`，新增段），并在自己判断该改时用 `update_outline` 整篇重写它。新结构、无历史消费方、无独立维护环节。
- **`update_outline` 工具** — 挂进续写的工具集（`world/tools.py`，与 update_world / notify / sense / sleep 并列），唯一调用方是续写 Agent 循环。
- **`world_deliberate` / `world_loop_instruction`** — 唯一消费方是续写（`engine.py:710/713`）。职责扩写后续写沿大纲的线推进客观世界、确立含贴角色的客观进程和身体状态、从大纲的线上克制地长出客观事件、自己判断何时回头改大纲、不碰主观。
- **续写 context 的 reminder 注入** — 在 `_world_loop_messages` 组装时，按 `WorldOutline` 最新版时间标注 + 当前是否 world 活跃时段，条件注入软提醒段。逻辑在续写侧，不新增环节、不新增钟。
- **`life_wake` prompt** — 消费方是 life 唤醒节点（`nodes/life_wake.py`）。加"world 客观断言不可否认、只决定主观回应"。这是原 spec Non-goals 漏掉的一面。
- **arc 的现有消费方不受影响** — 大纲独立、arc 语义不变，`arc_awareness.py` / `persona_review.py` / `eyes.py` / `reflection.py` / 续写读 arc 的行为全不变（独立 Data 相比扩 arc 的核心好处）。

## Data & deployment impact

- **新建 `WorldOutline` Data**（本次唯一 schema 变更）：自然语言整篇重写 + 时间标注 + append-only 版本链（结构同 `WorldArc`：`lane` Key、`narrative` 全文、时间标注字段、`version`）。走 framework 注册 + schema migrate；coe 冷启清库 + 旧表 fail-closed 注意（既往 coe 旧表删/改列被 migrate 整批回滚 pod crash 的坑）。新建表无删列改列，风险相对低。
- **Langfuse prompt**：`world_deliberate`、`life_wake` 改写（无新建独立 prompt——大纲维护并进续写、不单独成 prompt），prompt_vars 薄/无，先发 coe 泳道 label 验证、不直接动 production。
- **`update_outline` 的 durable/幂等**：大纲是 append-only 版本链、`update_outline` 是 durable mutation，而续写跑在 Agent.run 工具循环里——整轮 retry 会重放已执行的工具（既往教训：durable 工具必须 `max_retries=1` + 工具产物 id 从触发源派生幂等）。`update_outline` 要跟 `update_world` 同样处理，避免一次 retry 把大纲灌成重复版本。
- **agent-service 单服务部署**；部署杀 Pod、中断正在跑的后台世界推演——部署前确认无在跑任务或告知会中断。
- 先 coe 真机验证再上 prod；本次不改消息协议，MQ 遗留旧 schema 风险低。

## Tasks

1. **新建大纲 Data + `update_outline` 工具。** 目标：world 有一份可持久化的工作记忆——大纲，记世界此刻在走的几条客观线（每条"现在走到哪 + 客观接下来怎么走"）；续写工具集加 `update_outline`，让续写能读到最新版、并在自己判断该改时整篇重写它。产出：`WorldOutline` Data（append-only 版本链，结构同 `WorldArc`）+ `world/tools.py` 里 `update_outline` 工具 + 读取路径。验证：coe 建表成功、续写能写入并读回最新版大纲版本链。

2. **续写自维护大纲 + 沿线推进世界 + 职责重划。** 目标：续写每轮把大纲读进来当朝向、沿着每条线把客观进程往前推（含贴角色的就医流程出结果、落到身体的生理客观如发烧），从大纲的线上克制地长出客观规律下自然会发生的客观事件，全程不碰角色主观；world 自己判断何时用 `update_outline` 回头改大纲（自决低频），白天大纲长时间没更新时续写 context 注入软提醒引导它回看。产出：改写后的 `world_deliberate`（coe 泳道版）+ `world_loop_instruction` + `_world_loop_messages` 里大纲注入 + reminder 注入逻辑。验证：coe 构造绫奈就医（life 起头去医院），确认续写沿大纲的线把候诊/检查/出结果推到诊断出来、不再卡在"她在医院做检查"那一帧，event 从大纲的线长出不即兴硬造，不替绫奈写她怎么应对，且白天大纲长时间没更新时确认软提醒出现在续写 context（提醒后 world 改不改是它自决、不作为通过条件）。

3. **life 认账客观事实。** 目标：life 被唤醒读到 world 投来的客观事件时，客观部分（含身体状态）是既成事实、不否认，life 只决定怎么承受和应对。产出：改写后的 `life_wake` prompt。验证：coe 构造 world 投"她发烧到 39 度"，确认 life 那轮不演成"她觉得自己没事"，而是认账发烧、自主决定硬撑还是去医院。

4. **coe 端到端真机验证。** 目标：确认整套在 coe 撑住——剧情不再系统性停滞（不是单次观察，连续多日/多次统计停滞率，因停滞是概率性的）、多条线并行推进不丢、world 主动砸事件克制不淹没 life 的自主叙事、world 与 life 各守客观/主观不越界、成本与质量不随运行时长退化。验证：coe 连续跑数天，多条客观线（就医+等快递+学校流程+季节）并行往前走不卡死不丢线，输出里边界清晰、日成本受控、质量不降。
