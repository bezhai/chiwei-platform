# 赤尾世界 · 模拟一天

> 用三姐妹 + 原智鸿过完一个周二，把 event 驱动的 world/life 模型动态跑一遍。
> 静态模型见 `chiwei-redesign-world-model.md`，这篇是把它"开机跑起来"的版本。
> 不写技术实现，写"这一天怎么转、谁知道什么、为什么不卡死"。每个节点末尾用 **机制** 点出它在验证哪条设计。

---

## 先记住四件事

- **谁**：三姐妹是纯 AI 自驱的 life engine——千凪（chinagi，24，大姐，早起、厨房、管人）、赤尾（akao，18，老二、睡到下午、腹黑傲娇）、绫奈（ayana，14，老幺、初中生、起床困难）。原智鸿是特殊 life engine（默认 AI 脑补，真人随时能接管）。住在庆樾上府，楼下有绫奈的学校、便利店、巷子、超市。
- **world 管客观**：时间在流、谁在哪屋、世界冒了什么事、把"该谁感知的"投给谁。它懂这家人的作息节律（谁几点睡几点起、绫奈工作日要上学、八点上课），但它只铺客观情况、绝不下命令、绝不替谁定情绪。
- **life 管主观**：被 event 推醒 → 读自己当前状态 + 这批 event → 想一轮 → 决定干啥、更新自己 → emit 一个"我想做什么"的意图。她脑子里没有"做到几点"这种死时间，只有"此刻做什么"。
- **四类 event**：环境感知（在场才收得到，过期作废）、直连（指名给你、可跨空间、熟人才通）、地点痕迹（留在某处、谁路过谁看见、会被消耗）、意图动作（life 报给 world 去裁决的）。

## 这一天（周二，六月初）

### 05:50 开机

天还没亮，四个人都在各自房间睡着。world 启动时拿到一张起手快照：现在几点、四个人都在睡、各自是谁（人设）、家里什么布局。它的兜底心跳醒了一下，看了眼世界——这会儿没人动、没事发生，它没往外推任何东西，又安静下去。

**机制**：冷启动 = 起手快照 + 时间一走就转，不用凭空造第一个 event。平淡先沉成背景、不外溢，没事不烧 LLM。

### 06:10 千凪起床

world 时间推进到清晨，把"天蒙蒙亮了"这个客观现象投给浅眠的人。千凪本就是六点多自然醒的人，她的 life 被"清晨"唤醒，想了一轮：起，去做早饭、烧水手冲。她 emit 一个意图——"起床去厨房"。world 接住，裁定它发生了，把千凪的位置更新成厨房，记下"厨房开始忙活"。

**机制**：时间本身不是 event，是 world 醒来的理由；它得具象成"天亮了"才推给人。千凪靠自己人设接住"我该起了"，不是 world 命令她起。意图 → world commit 成客观事实。

### 06:30 厨房飘出味道

千凪做饭这件事，是她自己的事——但它甩出一个物理副产品：厨房和走廊里飘起煎蛋和咖啡的香味。world 接住这个客观现象，查这会儿谁能闻到：绫奈房门虚掩、半梦半醒，闻到了；赤尾房门关着、睡得死，闻不到。于是 world 只把"有早饭的味道"递给绫奈。

**机制**：做饭不是 event、飘出来的香味才是——动作的物理副产品才进通道。在场过滤天然制造信息差：赤尾收不到不是谁屏蔽她，是她物理上够不着。world 投的是"有早饭味"这个客观可感的形态，不是"千凪在疼你们"。

### 06:45 绫奈赖床

绫奈被香味唤醒，想一轮：再躺五分钟。她 emit"继续躺着"。注意她没有定"赖到 6:50"——她不设闹钟、不锁时间，"躺着"就是她此刻的状态。报完她就安静了，不再烧 LLM，等下一件事来敲她。

**机制**：她脑子里没有"时间段"，只有"此刻在干嘛"。赖床是她自己的决定，不是 bug。报完即安静，省成本。

### 07:05 千凪去叫她

千凪在厨房，心里装着"绫奈工作日得上学，这丫头三个闹钟都叫不醒"（她的人设）。她 emit 一个直连意图：朝绫奈房间喊"该起了，上学要迟到啦"。world 把这条直连投给绫奈（家里没别人在场旁听，赤尾还睡死）。绫奈这回被这条喊话唤醒，想一轮：行吧起了。emit"起床洗漱"。

**机制**：直连通道（指名给绫奈）。这就是"纠偏"——真有一个活人（千凪）注意到、来管，不是代码加了条"睡够 X 分钟强制起床"的规则。world 知道绫奈要上学，但它没去命令绫奈，是千凪基于自己"管妹妹"的人设去叫的。

### 07:35 绫奈出门

绫奈洗漱完出门。world 把她的位置一路更新到学校。开关门的声音是个环境现象，这会儿千凪在厨房听见了，world 投给千凪"绫奈出门了"。

**机制**：位置变更由 world 维护；动作的声响走环境感知，投给在场听得见的人。

### 08:00–08:40 上课

绫奈到校，emit"进教室上课"，当前状态变成"在上课"。这节课八点到八点四十，是学校定的、客观的——world 心里揣着这个节律。上课这四十分钟，绫奈安安静静、不被反复唤醒（除非同桌传纸条、或手机震了），到八点四十，world 推一条"下课铃响了"给她，她才醒来重新拿主意：去操场还是趴一会儿。

**机制**：有客观边界的事（上课），结束由 world 掌控、到点推 event，绫奈自己不记"上到几点"。上课期间她安静不卡死——这正是旧设计做不到的（旧的会锁死一个大状态干等）。

### 上午（大段平淡）

接下来的上午，这屋子基本是静的。赤尾睡到中午是她的常态；千凪做完家务、在阳台看她那几盆花，喝第二杯咖啡；绫奈在学校上下课。world 一直在内部走时间、推演世界，但它绝大多数时候的结论都是"这一刻不值得惊动谁"，于是这些都沉成世界当下的样子，不往外推。没人被没事找事地敲醒。

**机制**：平淡是常态，是"回头读世界时发现它安静地走过了几小时"，不是实时推送的流。防淹没第一靠 world 自己克制——大部分微动作压根不产 event。这里也顺带说明：赤尾睡到中午不该被纠——很多"睡很久"本就不是 bug，硬纠反而假。

### 12:30 赤尾醒

晌午的太阳晒进赤尾房间，world 推一条"光很亮、晌午了"给她，她醒了。她错过了一整个上午——但上午那些香味早过期了，没人给她补播，她就是没闻到。她迷迷糊糊走进厨房，撞见灶台上千凪给她留的饭（留饭是留在厨房的痕迹，她走到厨房才看见）；顺手摸手机，看到家庭群里千凪早上发的"早饭在锅里"（这条直连消息一直在那儿，她现在才读到）。这几样——晌午的光、留的饭、那条群消息——是攒在一起、她这一轮一次性消化的。

**机制**：环境感知过期作废、不补播；痕迹要到那个地点才撞见；直连消息异步（她醒了才读）。同一件"千凪管她吃饭"，通过三条通道、三种延迟到达。多条积压的 event 打成一批、一次想一轮（不是来一条醒一次）。

### 14:00 哥哥的私信

下午两点，原智鸿这一刻是真人接管的（他平时挂后台 AI 自驱，这会儿真人上线了）。他给赤尾发私信："晚上陪我去看楼下新开那家店，别跟妹妹们说。"这是对话，走的是 chat_loop 即时快路径——不绕 world、不夹一层思考，赤尾几乎立刻回了"好呀，几点？"。但这句话改变了世界（赤尾晚上要出门、还瞒着人），所以这条约定要回灌给 world 记一笔。千凪和绫奈对这一切毫不知情——她俩既不在这条私信里、也没在场。

**机制**：原智鸿是特殊 life + 真人随时干预。对话走 chat_loop 快路径求即时，不经 world。但对话里"改变世界的内容"必须回灌成 event，否则"聊天里的赤尾"和"世界里的赤尾"会变成两个人。私信是纯直连、无环境涟漪，妹妹们收到零个 event——信息差天然成立，没有一行"屏蔽"代码。

### 16:30 绫奈放学回家

world 把绫奈的位置推回家。她推门进玄关，开门声是环境现象，这会儿千凪在客厅听见了。假设赤尾这会儿已经在玄关留了张纸条"晚上不在家吃"——绫奈进玄关正好撞见，她读了，自己解读（可能有点吃味："凭什么不带我"，也可能无所谓）。要是赤尾还没留、或纸条留在了别处，绫奈这会儿就还不知道赤尾晚上要出门。

**机制**：放学到家由 world 推。痕迹靠"谁路过那个地点"触发，是最大的信息差来源。绫奈读完纸条怎么想，world 一个字都不插手。

### 18:00 晚饭

千凪把晚饭做好，喊一嗓子"吃饭啦"——这是广播给此刻在家、在场的人。赤尾这会儿正在房间换衣服准备赴约，她 emit"我不吃了，要出门"。千凪和在场的绫奈都听见了。千凪想一轮（基于残缺信息）："这丫头打扮成这样、不吃饭就走，是去见谁了吧？"——她脑补了一个版本，但她不知道是去见哥哥。她可能顺口问一句"去哪呀"，赤尾含糊带过。

**机制**：晚饭通知是环境广播、投给在场的人。信息差的妙处在这里：千凪绫奈现在知道"赤尾要出门"，但不知道"去见哥哥"——她们基于残缺信息各自脑补，可能补出错的版本。这就是戏。

### 19:00 赤尾出门赴约

赤尾出门，开关门声（环境，家里剩的人听见）。她在外面和哥哥待着的这段时间，绫奈憋不住发了条直连"姐你到底去哪了"，这条跨空间投到赤尾——但赤尾在约会、没顾上看手机（消息到了，她不在"看手机"的状态），就一直挂着，等她想起来翻手机才会看到。

**机制**：直连可跨空间、定向到人。但"消息送达"不等于"对方读到"——人不在那个状态，就产生延迟，又是一层信息差。

### 21:30 夜里的平淡

家里安静下来。千凪过了九点进入低电量（人设），回消息变慢、用词变短；绫奈画了会儿她那个会说话的猫的漫画，到点开始犯困。world 又是大段安静，没事不外溢。

**机制**：平淡 + 人设节律（同一个夜晚，各人各样）。world 不会为了"维持世界感"每隔几分钟硬塞一个 event——那会立刻退化回旧的每分钟卡死。

### 23:30 赤尾回家，留一张纸条

赤尾深夜回来，三个人都睡了。她想给哥哥留点什么，在玄关贴了张纸条"宵夜在冰箱，记得吃"，然后回房睡了。这张纸条挂在玄关，留的时候她并不知道谁会先看到、什么时候看到——也许明早绫奈出门撞见、也许哥哥先看到、也许根本没人看到。它就在那儿等着被撞见。

**机制**：痕迹的本质——接收方在留下那一刻是未知的，时间也悬而未决，还可能被人消耗或改掉（明早绫奈要是没看清、把纸条揉了、宵夜吃了，哥哥回来就什么都不知道）。这种长程、可能永远不被澄清的信息差，是即时通道给不了的余味。给第二天埋了个钩子。

## 这一天验证了什么

- **不卡死**：没有任何人锁死一个"做到几点"的大状态干等。每个人都是被一件件 event 推醒、当场重新拿主意——上课有客观边界（world 推下课），看书发呆没边界（靠下一个 event 打断）。
- **四类 event 都跑到了**：环境感知（香味、门声、晌午的光、晚饭通知）、直连（千凪喊绫奈、哥哥私信、绫奈追问）、痕迹（留饭、玄关纸条）、意图动作（每个人 emit 的"我想做什么"）。
- **信息差自然涌现**：千凪不知道赤尾去见谁、绫奈靠纸条迟知道、哥哥可能永远不知道纸条被截胡——全是"谁在场、谁认识谁、谁路过哪"这几个客观事实叠出来的，没有一行"控制可见性"的代码。
- **纠偏是内生的**：绫奈起不来，是千凪去叫（真有人管）；万一全家都反常地不动，world 当导演会加戏。不是加规则检测异常。
- **平淡没有淹没谁**：大段时间 world 安静、沉成背景，只有够格的事才惊动人。

## 还没抠死、先做出来再调的几个度

- **感知颗粒度**：同屋里多小的动作该传给在场的人、多小的不传（免得噪声淹没）。
- **当面说话的双重身份**：当面说一句话，既是说给某人的直连、又是同屋别人听得见的环境感知（还可能听不清）——这块先做最朴素版跑跑看。
- **原智鸿脑补尺度**：真人离线时，AI 替"哥哥"自驱到什么程度。
- **节奏参数**：world 心跳多密、life 攒批等多久（debounce 窗口）——这些是参数，跑起来调，不在设计阶段拍死。
