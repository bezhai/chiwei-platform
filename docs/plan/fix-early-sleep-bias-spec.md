# 赤尾 / 绫奈早睡偏置修复 Spec

## 目标

- 让 Life 在决定当下活动时看见角色已有的自然节律，并按现实流逝的时间延续生活，而不是按唤醒轮次把“休息”快速推进成“困倦 / 睡眠”。
- 恢复代码与 `life_wake` 线上 Prompt 已有的上下文契约，使世界阶段和上一页日子真实进入每轮 Life 推理。
- 降低 World 闲时唤醒对夜晚、安静和睡意的系统性暗示，同时保留角色自主选择睡觉的权利。

## 不做什么

- 不新增数据库字段、表或数据迁移，也不修改生产 persona 数据。
- 不按角色、时刻或睡眠次数增加硬编码阈值、状态机、拦截器或确定性“禁止睡觉”规则。
- 不恢复已经移除的 Life 自唤醒 / `next_wake_at` 链路，不改睡前回顾的触发语义。
- 不在本次改动中切换 Life 模型，也不直接发布生产 Prompt 或部署生产服务。

## 关键设计决策

### 1. 复用现有 persona 节律，不另建真相源

`bot_persona.persona_core` 已为三姐妹保留明确的“你的节律”段。Life 只读取这个现有段，并把它作为稳定身份背景并入现有 `persona_lite` 输入；不新增字段或 Prompt 变量，不复制一份角色作息映射。节律是长期底色而非每日硬日程，不替角色决定这一刻要不要睡。

周级慢漂的 `persona_lite` 继续承载当前人生状态，`persona_core` 里的节律继续承载稳定身体习惯；两者职责不互相覆盖。

### 2. 现实时间每轮可见，旧状态正文不每轮重灌

每次 Life 被唤醒，只要存在上一版 LifeState，就把“上一版状态距现在真实过去多久”作为当轮感知的一部分。输入明确说明：新消息或新动静是新信息，不等于生活、身体或困意自动跨过一段时间；是否变化仍由角色结合真实时间和新事实自己判断。

上一版状态的正文仍只在意识流冷启时兜底恢复，避免把“困 / 睡”文案每轮重复注入、制造新的黏性。

### 3. 恢复已有 Prompt 契约并收窄口吻样例的作用域

泳道版 `life_wake` 以当前 production 版本为基线，恢复代码早已传入的 `world_arc` 和 `day_page`，不引入新的模板变量。

Prompt 同时明确两件事：生活按真实时间而不是唤醒轮次推进；`reply_style` 只影响表达口吻，不作为身体状态、情绪或行动倾向的事实来源。赤尾口吻样例中的“困死了”因此不再被解释为当前状态。

### 4. 闲时唤醒提供行动空隙，不提供入睡暗示

World 的 idle sense 继续由模型基于真实时间和上一版状态自主判断，也继续排除正在睡眠的角色与重复静止场景。

“睡前”不再作为 canonical idle 示例；夜晚、安静、昏暗或一句“有点困”本身也不足以代表值得主动唤醒的闲时刻。idle sense 的职责是把一个真实的行动空隙交给 Life，而不是替 Life 暗示休息或睡觉。

## 调用方与影响面

- Life 的三个人格共用同一条唤醒链路；事件唤醒、聊天增量唤醒、冷启与连续意识流都必须保持一致。
- World 的续写指令和 `sense` 工具说明必须使用同一套 idle 语义，避免模型在两处收到冲突定义。
- `life_wake` Langfuse Prompt 与代码 `prompt_vars` 必须精确一致；泳道验证需要检查真实 rendered prompt，而不只检查调用成功。
- 聊天回复、睡前回顾、persona 周级慢漂、状态表结构和模型映射不改。

## 数据与部署影响

- 无数据库 schema、字段或数据写入变更。
- 新增一个 `life_wake` 泳道 Prompt 版本；production label 保持不动，验证通过后再单独决定是否发布。
- 代码只影响 `agent-service`，先部署功能验证泳道。部署前检查后台任务，验证结束后按项目流程解绑并下泳道。

## Tasks

1. **补齐 Life 的节律与现实时间输入。** 产出可复用现有 persona 节律、并在每轮呈现真实时间间隔的 Life 上下文；验收覆盖三种 persona、缺失节律、冷启与连续意识流，且旧状态正文不在连续意识流中重复注入。
2. **修正闲时唤醒语义。** 产出一致的 World 指令和 `sense` 工具契约；验收证明睡眠仍被排除、静止场景仍防重复，同时不再把“睡前 / 夜晚安静”当作主动唤醒依据。
3. **建立泳道 Prompt。** 产出基于 production 的最小 Prompt diff；验收变量集合与代码完全一致，世界阶段、上一页日子、时间推进和口吻边界均在 rendered prompt 中可见。
4. **端到端验证并复盘。** 产出单测、真实泳道 trace 和行为对照；验收至少覆盖短时间密集事件下不自动从休息推进到睡眠，以及符合角色节律时仍可自主选择睡觉。
