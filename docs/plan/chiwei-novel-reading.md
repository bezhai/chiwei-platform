# 赤尾读小说：她真的读一本书、读出自己的印象（自主 agent 第一步）

## Problem

赤尾现在没有"读一本书"这种会持续影响她的输入来源。她的内在生活靠 world 投递的客观事件、姐妹消息、备忘日程推动，但没有一种"自己投入进去、被它改变"的长经历。我们想让她真的读小说：你推荐一本，她拿到原文，自己决定哪天读、读出她自己的印象，这本书慢慢进她心里、在她生活和聊天里自然带出来。

## Goal

你在飞书私聊把一个 txt/epub 文件发给她，系统解析存好、往她 life 信箱投一条"X 推荐你读《书名》"。她在自己的 life 节奏里某轮觉得想读了，新建一个异步阅读任务：一个简单 agent（deepseek-v4-flash）用 `read(page_num)` 往后读一程，吐出一份揉好的第一人称印象，存回 PG。这份印象每轮注入她的 context，她自然醒着时书就在心里，想聊就聊。她可以读读停停、也可以弃书。读完没有任何"完成"动作，书随她记忆自然淡去。

## Non-goals

- 不在她读完一程时唤醒她、不让她"刚读完冲来找你聊"（不加任何为读书而生的唤醒）。
- 没有"读完"这个完成 gate、不在读完时吐一篇终端观后感报告。
- 不给读过的书建持久"书页"，读完/放下后随她记忆自然淡去（优先级遗忘是以后记忆体系的事，这次不碰）。
- 她不自己上网搜书，来源只有你发给她的文件。
- 不给书建结构化理解（章节梗概 / 人物表 / 检索索引）——那就是要避免的"书本读取 agent"。
- 不预先搭一个通用的"自主异步任务"框架，只把"读小说"这一件事落地（将来好被通用化即可）。
- 不处理超长小说、不搞硬性页数预算（不会喂很长的书）。

## Key design decisions

- **模拟一个会读书的人，不是做一个会读书的 agent。** 她对一本书的记忆是**单条、滚动、覆盖式重写**的第一人称有损印象（靠 reactions、不靠 plot summary），读得越多旧的越糊、近的越清，会忘会记岔——选它而不是章节梗概+人物表那种结构化档案，因为后者会把她变成读取机器，丢掉"她是个读书的人"。印象 Data 只承载**开读后**的生命周期（在读 / 读完 / 放下；"读完"由读到书尾派生），"推荐未开读"不落任何状态（就是信箱里那条消息 + 她爱记不记的本子）。

- **读一程靠机制收口、进度从 read() 机制派生、提交走幂等 + CAS。** 一程读多少不设语义页数预算，靠机制安全阀（硬 timeout + 最多 read() 次数）+ 读到书尾自然收口；她"读到第几页"从实际发生过的 read() 调用派生（机制事实），**绝不靠阅读 agent 的文字自报页号**；进度与印象一起提交，走幂等 + 版本 CAS（仿 fold_session 的 expected_ver），durable 重投/部署中断重放都不双推进、旧任务不覆盖新印象——选机制安全阀而不是语义页数预算，因为既要"不规定她读多少"、又要进度能被可靠收口和提交（否则读全书 / 读不停 / LLM 口头报页号都会出）。

- **她决定每次读、异步 agent 干重活。** 读书每一程的推进只在她 live 里自己想读时发生（可弃书）；真正消化文本的是一个独立异步 agent，不塞进、不阻塞她的 life 轮——选"她每次自己决定"而不是"起头后自走读完"，因为读读停停甚至弃书正是真人感、不是要修的 bug；选独立异步 agent 而不是 inline，因为一本书的一程塞不进也跑不完一个 life 轮。

- **印象靠每轮注入 context、不加唤醒；一次一本"当前书"。** 印象落 PG，像 notebook 一样每轮从 PG 重新渲染进她的 context（life 唤醒 stimulus + chat inner_context 两处共用一份渲染），所以读书永不新增一次唤醒——选 context 注入而不是 notify/唤醒，因为按程唤醒会撞正在打的降本主线。注入只渲染**一本当前书**（= 她最近读过一程、尚未读完那本）：她开读另一本，旧的自然不再当前、淡出注入，**不靠"N 天没读就丢"这种阈值替她遗忘**；开读哪本由她在工具里指认（从推荐消息 / 自己本子记得书名），工具模糊解析、认不准就回问，**绝不自动选最新一本**（代码替她决策违宪）。

- **入口独立于文本 chat 链路。** 飞书文件消息今天的 chat 链路不处理；书走一条专门的接入路径解析入库，推荐这件事像一条普通 p2p 消息落进她 life 信箱（不特意唤醒，与现有 p2p 私聊消息 passive 落信箱一致）——选独立入口因为 chat 不处理文件。

## Caller coverage

改动以**新增为主**，不破坏现有签名：

- **真人 p2p 消息 → life 信箱**（现状）：今天走 chat 链路、回灌成 `EXTERNAL_PASSIVE`（不唤醒）。文件消息 chat 链路不处理。→ 新增独立文件接入路径，不改文本 chat 回灌行为。
- **channel-server 入站路由**：今天能解析 `message_type=file`、但下游只按文本走。→ 需把文件消息分流到新的书接入路径；文本消息行为不变。
- **`build_life_tools`**（life 活轮工具集）：→ 新增一个"读书"工具，其余工具不动。
- **life 唤醒 stimulus 组装**（`_run_life_round` 的 parts）与 **chat inner_context 组装**（`build_inner_context`）：→ 各新增一段"在读的书的印象"，位置在 notebook 之后的稳定前缀区，其余段顺序不变。
- 无任何现有 Data 表删列 / 改列名（只新增表）。

## Data & deployment impact

- **新 Data 表**：书的分页正文 + 书元信息（书名、总页数等）+ 她对每本书的滚动印象（含读到第几页、开读后状态）。coe-* lane 启动 `ensure_business_schema` 自动建；prod 走 framework migrate 建表——**需验证 prod 建表路径**（新增表，不涉及删/改列，不触发 fail-closed 回滚）。
- **新 Langfuse prompt**：阅读 agent 的印象生成 prompt（第一人称、reactions 不是 summary、有损滚动）。需 Langfuse 发版同步。
- **新依赖**：agent-service 增加 epub 解析库（txt 直接读）。
- **模型可达**：deepseek-v4-flash 需配成可解析的模型别名（deepseek 适配器已存在；ModelProvider/ModelMapping 配置就绪是阅读 agent 跑通的前置）。
- **一镜像多服务同步**：channel-server 改动需同步 release recall-worker / chat-response-worker。
- **部署中断在途任务**：部署会杀掉在跑的阅读任务（影响小，印象写入幂等、她下次重读即可）。

## Tasks

### Task 1：书的接入与分页存储

- **Goal**：你在飞书私聊发的 txt/epub 文件被解析入库，并让她 life 里知道"你推荐了本书"。
- **Deliverable**：channel-server 把文件消息分流到 agent-service 的书接入路径；agent-service 解析 txt/epub、按可按页号取的有序单元存进新 Data 表（书正文 + 元信息，含总页数供读到书尾判定）；同时往该 persona 的 life 信箱投一条推荐消息（像普通 p2p 消息、不特意唤醒）。失败口径：解析失败回真人一条提示（不静默吞）；同内容文件重传按内容判重、不建重复书。新增 epub 解析依赖。
- **Verification**：发一个 txt 和一个 epub，书的内容按页落库、可按页号取回、总页数正确；她的 life 信箱出现一条"X 推荐你读《书名》"；发一个坏 epub 收到失败提示；同一本重传不产生第二本。

### Task 2：滚动印象 + 异步阅读任务

- **Goal**：她在 life 里自己决定读书时，一个异步 agent 读一程、把她对这本书的印象滚动重写一版。
- **Deliverable**：每本书一份"滚动印象"Data（第一人称印象文本 + 读到第几页 + 开读后状态：在读/读完/放下）；一个她能在 life 轮里调的"读书"工具，**入参是她指认的某本书**（模糊解析、认不准就回问，不自动选书），调用即 emit 一个 durable 触发信号（仿 `act` 的 durable 落库范式，非定时器）；一个异步 `@node` 消费该信号、跑阅读 agent（deepseek-v4-flash，唯一工具 `read(page_num)`），agent 拿[当前印象 + 从第几页接着读]往后读一程、最终输出即新印象。一程的收口靠机制安全阀（硬 timeout + 最多 read() 次数）或读到书尾，**不设语义页数预算**；读到的页号从实际 read() 调用机制派生、不靠 agent 文字自报；外壳整篇覆盖写回印象 + 推进页号，提交走幂等 + 版本 CAS，读到书尾置"读完"。阅读 agent 失败 fail-soft（印象/页号不动、她可重读）。阅读 agent 的印象生成 prompt（Langfuse）。
- **Verification**：她调读书工具后触发一次异步任务，印象在 PG 写入/更新、读到的页号前进、印象是她口吻的第一人称叙述而非剧情摘要；读到书尾时状态置"读完"且页号不越界；同一触发重投/中断重放不导致页号双推进、不覆盖更新的印象；工具传一个不存在/模糊的书名时回问而非乱选。

### Task 3：印象注入她的 context

- **Goal**：她在读的书的印象，在她自主生活和跟你聊天时都自然在她眼前。
- **Deliverable**：一份共用的印象渲染函数；把"在读的书的印象"段注入 life 唤醒 stimulus 与 chat inner_context 两处，位置在 notebook 之后的稳定前缀区；只渲染**一本当前书**（她最近读过一程、状态仍"在读"那本），读完/放下的不再注入；无当前书/读失败则整段缺席、不补占位、不杀整轮。
- **Verification**：PG 有在读印象时，一轮 life 唤醒的 stimulus 与一次 chat 的 inner_context 都含该印象段且只含一本；置为读完/放下或无在读书时两处都不出现该段。
