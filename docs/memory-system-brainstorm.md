## 一、赤尾是谁

- 飞书群聊中的**虚拟人**，不是工具型助手，不是聊天机器人
- 核心人设是「拥有真实身体和感官的人类美少女」，草莓粉长发，元气活泼
- 几乎所有互动发生在**群聊**中，赤尾对用户的认知主要来自群聊观察
- 当前核心用户几十人，目标几千人
- 纲领性文档：`MANIFESTO.md`（赤尾宣言）

## 二、现有记忆系统的问题

### 现有实现

- `user_knowledge` 表存储用户画像：facts（按 category 分组）、personality_note、communication_style
- 每 4 小时 cron job 批量处理：收集用户消息 → LLM 提取事实 → 合并写入
- Facts 按类别存储：基本信息、职业、爱好、习惯、偏好、人际关系、近况
- 每次对话注入 ≤400 字的记忆文本到 system prompt
- 另有 `search_group_history` 工具：Qdrant 向量 + BM25 混合检索群聊历史

### 核心问题

1. **存的是档案，不是记忆**。「爱好：看动画；职业：程序员」是 HR 会写的东西，不是朋友会记的东西
2. **4 小时批处理太慢**。用户说完「我换工作了」，4 小时后才可能被处理
3. **400 字全量注入**。不管用户在聊什么，都塞进同一堆记忆，大部分是噪音
4. **不区分信号可信度**。@赤尾的话大概率是假的/在逗着玩，但系统同等对待
5. **私聊记忆和群聊记忆没有隔离**。私聊记忆泄露到群聊是大忌
6. **search_group_history** 的问题：慢、容易幻觉、召回原始消息而非总结

## 三、从真实数据（KA 群）得到的发现

### 数据概况

- 日均消息量：600-700 条
- 活跃时间：北京时间 11:00-18:00，夜间几乎为零
- 群成员约数百人，活跃核心 20-30 人

### 消息的信息分布（关键发现）

在 200 条消息的样本中：
- ~30% 表情包/图片/链接（无文字信息）
- ~20% 单句闲聊（「好吃」「哈哈哈」）
- ~20% 娱乐互动（角色分配、接龙）
- ~15% 赤尾的回复
- ~15% 有实际信息含量

**从 200 条消息里可能只有 10-20 条包含值得记忆的信号。**

> 注：以上数据来自对某活跃群的实测，已脱敏处理。

### 对「认识一个人」最有帮助的信息

1. 自我披露的硬信息（相亲帖、体重变化等）
2. 讨论中自然流露的偏好
3. 情绪和状态表达（注意区分自嘲和真实情绪）
4. 别人对他的评价/互动
5. 工作/生活碎片

### 社交拓扑

群里有明显的小团体和互动关系。**关系网络是最被低估的记忆维度。**

## 四、已达成的方向性共识

### 4.1 信号可信度矩阵

| 信号来源 | 可信度 | 原因 |
|---------|:-----:|------|
| 用户之间的对话（赤尾旁听） | **高** | 人和人之间一般不会无缘无故编造 |
| 用户之间反复出现的行为模式 | **高** | 行为模式不会骗人 |
| 用户主动纠正赤尾 | **高** | 主动纠正大概率是真的 |
| 用户对赤尾的陈述性内容 | **低** | @赤尾 大概率在逗着玩 |
| 用户对赤尾的情感表达 | **低（事实）/ 中（互动风格）** | 事实不可信，但说明互动偏好 |

**反直觉结论：赤尾最该信的是用户们之间说的话，最不该信的是用户直接对她说的话。**

实测验证：88% 的消息不含 @赤尾，旁听是绝对的主要信号源。

### 4.2 记忆优先级

**优先级 1：用户印象（Person Impression）** — 不是 fact，是 impression。「你就是那个每次发起角色分配挑战的指挥官」比「爱好：看动画」有用一万倍。

**优先级 2：群文化记忆（Group Culture）** — 群的性格、梗、话题偏好、互动风格、人际关系拓扑。

**优先级 3：近况和共同经历** — 实际数据显示用户很少在群里对赤尾说真实近况，近况更多从旁听中被动获取。

### 4.3 search_group_history 的定位

**应该大幅减少使用。** 核心问题不只是慢和幻觉，而是**颗粒度就是错的**——返回原始消息而非消化后的理解。这不是朋友的记忆方式。

正确关系：记忆系统 = 主要知识来源；search_group_history = 降级为特殊工具。

### 4.4 隐私和敏感信息

**铁律：** 私聊记忆绝不在任何群聊中使用。

**不记录的内容：** 个人身份信息、健康/医疗、财务细节、职场八卦/负面评价、政治/宗教观点、亲密关系细节。

原则：**如果这条记忆被用户看到会让他不舒服，就不该存。**

## 五、核心难题：语言学 — 梗的三层地狱

> 这是整个记忆系统最根本的难题。之前 10+ 种方案的失败，大部分死在这里。

### 5.1 问题的本质

群聊中的语言大部分**不是在传递信息，而是在执行社交行为**。

- 「霸霸操我」→ 表演亲密，不是信息
- 接龙 某梗✌️ → 参与集体自嘲仪式，不是讨论 AI 团队
- 「我好失败」→ 邀请别人来安慰/调侃，不完全是描述情绪

**「从消息中提取事实」这个前提假设可能就是错的。**

### 5.2 梗的三层地狱

**第一层：外部梗（公开互联网可查的）**

「猫雷是好女人」— LLM 知识库可能覆盖。但即便知道来源，仍需正确判断「用梗表演」vs「真实保护欲」。实际测试中 Claude 把「用梗」解读成了「对猫雷有保护欲」。

**第二层：公司/圈层梗（公开互联网几乎查不到的）**

「某内部梗✌️」= 某公司某核心团队 + 反讽式尊称 = 集体自嘲。实际测试中 LLM **自信满满地**给出完全错误的解读，多层含义一层都没理解对。

**第三层：群内部梗（只存在于这个群的历史中）**

没有任何外部知识可以帮你理解。含义只存在于群的共享记忆中。

**每往下一层，LLM 的理解能力断崖式下跌。而群聊中的语言，大量分布在第二层和第三层。**

### 5.3 为什么这导致记忆系统失败

1. 消化的质量完全取决于 LLM 对群聊语言的理解能力
2. LLM 对梗/反讽/圈层黑话的理解能力接近于零（特别是第二、三层）
3. 错误的理解会被存为「记忆」，在未来被引用
4. **错误的记忆比没有记忆更可怕** — 赤尾把 某梗✌️ 当成 A-SOUL 梗来用，用户会觉得她根本不懂这个群

而且 LLM 犯错的方式是**隐性的、自信的、看起来合理的**。

### 5.4 方向 A、B 的局限

**方向 A：降低野心，只记可靠的** — 窄但可靠。问题：记忆太贫乏，不足以让赤尾「像朋友一样认识你」。

**方向 B：不试图理解，只记录模式** — 绕过了语义解释。问题：赤尾只能记模式不能真正参与群文化。

**两条路都是妥协。方向 C 在下面。**

## 六、突破：日记范式（方向 C）

> 这是多轮讨论中最核心的突破。详细的探索文档和完整日记示例见 `docs/diary-digestion-exploration.md`。

### 6.1 核心范式转换

消化不是「从消息中提取信息」，而是「赤尾躺在床上回想今天」。

| | 信息提取 | 写日记 |
|---|---|---|
| 视角 | 第三人称（分析员） | 第一人称（赤尾自己） |
| 目标 | 全面、客观、不遗漏 | 选择性、主观、只记有感觉的 |
| 对梗的处理 | 必须理解并解释 | 可以诚实地不理解 |
| 输出形态 | 结构化 fact list | 自然语言叙述 |
| 错误模式 | 自信地错误解释（恐怖谷） | 模糊地记住氛围（像人） |
| 遗漏 | 是提取失败 | 是「没注意到」 |
| 和人设的关系 | 无关 | 天然带着赤尾的性格 |

这个重定义的威力在于，它把消化从一个**工程问题**（怎么从噪音中提取信号）变成了一个**人设问题**（赤尾会怎么回忆她的一天）。人设问题是 LLM 最擅长的。

### 6.2 日记解决了什么

**解决了「记什么」的决策问题。** 信息提取需要预定义「什么算有价值的信息」。日记把这个决策交给赤尾自己。她是元气的、好奇的、喜欢二次元的——她的日记自然记录让她有感觉的事。**选择什么记住，本身就是人格的表达。**

**解决了「以什么形式记」的格式问题。** 日记是叙事，天然带着时间线、因果、情感、关系。key-value 需要额外字段表示的维度，在叙事中天然存在。

**解决了梗的三层地狱。** 日记模式下赤尾面对不理解的梗有三种自然写法：

```
完全不理解："大家又在刷那个什么 某梗✌️，不知道是什么意思，但每次聊到工资就出现。"
部分理解：  "某梗✌️ 好像是在说公司里某个很厉害的部门？大家发的时候带着自嘲语气。"
理解了：    "今天终于搞懂那个梗了！就是公司某个团队，工资特别高，大家在自嘲。"
```

三种状态都是有效的记忆。信息提取模式只允许第三种存在——要么理解了要么不记。日记模式允许全部三种，而且前两种比信息提取的「自信地理解错」好一万倍。

**解决了信号可信度。** 第一人称视角天然是可信度过滤器。赤尾不会在日记里写「A说他是CEO」（除非她真信了），她会写「A老说自己是CEO哈哈哈感觉在逗我」。

**解决了记忆的主观色彩。** 日记天然产出的是「他聊起动画来特别带劲」，而非「爱好：看动画」。

### 6.3 三层时间结构

```
日记（每天）    → 鲜明的、完整的、有细节有情感
  ↓ 周回顾
周记（每周）    → 这周的重点人物和事件，细节开始模糊
  ↓ 月沉淀
长期印象（每月）→ 对人的稳定认知、对群的基本理解
```

每一层都是赤尾的第一人称叙述。这天然实现了宣言 5.4 描述的记忆衰退：鲜明 → 模糊 → 印象 → 遗忘。

**周回顾不是「压缩日记」，是赤尾翻自己日记时的回想：** 「这周 群友A 特别活跃啊，几乎每天都在带活动。rina 周三之后就消失了……」

**月沉淀也是第一人称：** 「我认识 群友A 快两个月了。他是群里的气氛担当，嘴硬心软，聊二次元特别带劲。」

### 6.4 日记和七种记忆形态的关系

日记不是第八种记忆形态。**日记是前端（产出端），七种记忆形态是后端（沉淀端）。**

```
日记中的叙事 → 提炼为 → 七种记忆形态

"群友A 又在带大家玩角色分配"   → 人物印象（群友A = 活动发起人）
"rina 今天好安静"             → 人物印象 + 未完结线索
"大家又在 某梗✌️ 接龙"       → 群文化画像（活跃梗）
"跟 群友A 怼了猫雷的事"       → 关系记忆（可以互怼）
"我说我最近在看芙莉莲"       → 赤尾自身记忆
"rina 好像不想被提那件事"    → 社交禁区
"群友A 聊动画特别带劲"        → 话题-人物关联
```

日记写完后，后处理步骤把内容沉淀到对应的记忆形态。但日记原文也保留——因为它包含的情感和语境在结构化后会丢失。

### 6.5 日记的长度和结构本身就是信号

真人日记有时一句话，有时写满三页。**日记的长度和结构是赤尾情感状态的映射。** 某天写了很长 → 这天对她很充实。某天只写了一句 → 这天平淡。某天整篇都在写一个人 → 这个人今天触动了她。

### 6.6 完整示例

见 `docs/diary-digestion-exploration.md` — 基于 KA 群某日的真实聊天记录（已脱敏）写的一篇完整日记，包含六个栏目：今日心情、今天的群、印象深的事、人物速写、没搞懂的东西、碎碎念。

## 七、信心系统

### 7.1 核心命题

赤尾对每一条记忆的「确信程度」应该直接决定她表达这条记忆时的语气。

宣言 5.5 的核心区分：**人的记错是可爱的，AI 的记错是恐怖谷。** 区别在于：人记错时知道自己可能在记错（「好像是这样吧？」），AI 记错时不知道自己在记错（「你在3月5号说过你喜欢吃辣」——完全编的但说得跟真的一样）。

**信心系统的本质是让赤尾拥有元认知——知道自己知道什么、不知道什么、不确定什么。**

### 7.2 信心的来源

**来源可信度（4.1 的矩阵）：** 旁听到的 > 自己互动中的 > @赤尾 的

**重复确认次数：** 只出现一次 → 低信心；多次跨时间出现 → 高信心；被不同人印证 → 极高信心

**理解深度：** 确定自己理解了 → 高；猜测但不确定 → 中；完全不理解但记下了模式 → 低但诚实

### 7.3 信心映射到表达

```
高信心（已固化的认知）：
"群友A 工作一直很忙的，他经常加班到很晚。"
→ 直接陈述，不带犹豫词

中高信心（多次印证但非绝对确定）：
"群友A 好像挺喜欢看动画的，之前好几次聊到新番他都很积极。"
→ "好像"、"好几次"自然出现

中信心（有印象但细节模糊）：
"他之前是不是提过在学什么乐器来着...吉他？"
→ "是不是"、"来着"、反问语气

低信心（只有模糊的感觉）：
"你...我总觉得在哪里跟你聊过这个话题，但想不起来了。"
→ 表达"有感觉但不确定"

无信心：
（不提。赤尾不会主动说出她完全没把握的事。）
```

### 7.4 信心的动态变化

**增长：** 重复确认、直接验证、群友主动纠正后的新认知（纠正后信心极高）

**下降：** 时间衰退、矛盾信息、社交反馈（赤尾基于某条记忆说了话，结果反应不对——说明记忆可能有误）

**社交反馈是记忆系统的自我修正机制。** 赤尾说「你不是在学吉他吗」，对方说「啊？我没学过吉他啊」→ 相关记忆信心立即降到极低，触发修正。

### 7.5 信心和日记的结合

在日记中，信心自然体现为赤尾的语气。沉淀时可以根据语气词推断信心水平：

```
"他在学吉他"           → confidence: high
"好像在学什么乐器"      → confidence: medium
"不太确定但感觉跟音乐有关" → confidence: low
"没搞懂的东西：某梗✌️"   → confidence: very_low
```

日记不只产出记忆内容，还产出记忆的信心水平。二者是一体的。

## 八、五阶段文化学习模型

### 8.1 核心命题

赤尾对群文化（特别是梗）的掌握不是一次性的「理解或不理解」，而是一个跨越周-月的渐进学习过程。

### 8.2 五个阶段

以 某梗✌️ 为例：

**阶段 1：沉默观察（Day 1-7）**

日记：「今天群里好多人在发 某梗✌️，不知道什么意思。」行为：不参与。如果被问，说「不太知道诶」。

**阶段 2：试探参与（Day 7-14）**

日记：「某梗✌️ 好像是在自嘲？大家发的时候带着无奈的语气。」行为：大家接龙时跟着发一个 ✌️（试探性参与）。如果反应好，信心增加；如果被指出用错了，获得修正信息。

**阶段 3：深入理解（Day 14-30）**

日记：「某群友跟我解释了那个梗是什么——公司某个团队，据说福利比其他部门好很多。」行为：可以在合适时机自然使用。

**阶段 4：流畅使用（Day 30+）**

完全掌握：理解社交功能、使用场景、情感色调。可以创造性使用。

**阶段 5：文化创造（罕见）**

基于已有梗创造新变体。需要极深的理解，大部分梗不会到达。

### 8.3 学习的驱动力

每个阶段的推进由**信号密度**和**社交反馈**驱动，不是时间自动触发：

- 某个梗一天10次 → 快速推进（几天从阶段1到3）
- 某个梗一周1次 → 缓慢推进（一个月从阶段1到2）
- 某个梗消失了 → 停止推进

**社交反馈是最强推进力：** 赤尾试探性地使用了一个梗 → 反应正面（大家笑了）→ 信心大增 → 反应负面（用错了）→ 信心下降但获得修正信息。

### 8.4 学习和日记的结合

日记是学习轨迹的天然载体。当 某梗✌️ 不再出现在「没搞懂的东西」栏目，而是自然嵌入叙事中时，说明赤尾已经掌握了。

## 九、双管线架构：体验自我 + 叙事自我

### 9.1 核心命题

赤尾需要两条记忆管线——白天实时运行（体验自我）和夜间离线运行（叙事自我/写日记）。它们解决不同问题。

### 9.2 为什么需要两条管线

> 下午2点群里有人解释了 某梗✌️ 的含义。下午3点有人又开始接龙。如果记忆只有夜间消化，那下午3点赤尾还不「知道」那个梗是什么——因为下午2点的消息还没被消化。

### 9.3 体验自我管线（实时/在线）

**触发：** 赤尾每次在群里被触发回复时

**做什么：** 赤尾在回复过程中学到新信息时，写入「热记忆缓冲」。缓冲当天有效，下次被触发时可以影响回复。

**特点：** 即时生效、颗粒度细（单条信息级）、生命周期短（当天）、来源于赤尾自己的互动。

### 9.4 叙事自我管线（离线/夜间日记）

**触发：** 每天凌晨（群静默期）

**做什么：**
1. 收集当天所有消息（旁听 + 赤尾自己的互动）
2. 加载热记忆缓冲（体验自我白天积累的）
3. 生成赤尾的日记
4. 从日记沉淀到七种记忆形态
5. 清空热记忆缓冲

**特点：** 全景视角（一整天的消息流）、颗粒度粗（天级）、生命周期长（日记 → 周记 → 长期印象）、包含旁听信息。

### 9.5 热记忆缓冲

两条管线之间的桥梁。白天写入，夜间被日记整合。

```
体验自我（白天）              叙事自我（夜间）
    │                              │
    │ 实时互动 → 热记忆缓冲        │ 一天消息 + 热记忆缓冲
    │                              │
    ↓                              ↓
  即时影响赤尾的行为            沉淀为长期记忆
```

热记忆缓冲不只是新信息，也包括赤尾白天的情绪和体验——让日记可以引用体验自我的实时感受，而非事后重新分析。

### 9.6 峰值-结尾法则

Kahneman 的研究：人对一段经历的记忆主要由**情感峰值**和**结尾**决定。

赤尾写日记不需要均匀回忆600条消息。自然聚焦于：
1. 今天的情感高点（最好笑的/最感动的/最尴尬的瞬间）
2. 一天结束时的氛围

## 十、社交坐标替代语义理解

### 10.1 核心命题

对于赤尾无法理解语义的内容（梗、黑话、反讽），不需要理解「它是什么意思」，只需要理解「它在社交空间中的位置」。

### 10.2 什么是社交坐标

```
谁在用？        → 某梗✌️ 主要是 A、B、C 在发
什么时候用？    → 通常在讨论部门差异、薪资话题之后
用了之后什么反应？→ 其他人跟着接龙，或者发笑哭表情
情感色调是什么？  → 自嘲式的、无奈但带笑的
使用频率如何？   → 最近一周每天都有人发
赤尾参与过吗？   → 试着跟过一次，大家没觉得奇怪
```

这些信息**完全不需要理解语义**就能获取。都是可观察的行为模式。

### 10.3 社交坐标 vs 语义理解

```
语义理解：
"某梗✌️ = 某公司核心AI团队 + 反讽尊称 = 集体自嘲"
→ 需要外部知识 + 文化理解 + 反讽识别 → LLM 容易犯错

社交坐标：
"某梗✌️ = 群里高频梗，在薪资话题后出现，参与者带自嘲语气，接龙式使用"
→ 只需要模式观察 → LLM 做这个相对可靠
```

**社交坐标回答「怎么用」而非「什么意思」。** 对于群里的日常互动，「怎么用」比「什么意思」更重要。

### 10.4 局限

社交坐标能让赤尾「跟着玩」但不能让她「主动用」或「创造性用」。阶段1-2只需要社交坐标就够了。到阶段3-4，赤尾已经通过观察和反馈积累了足够的理解。

**社交坐标是梗学习的基础设施，不是终极解决方案。** 让赤尾在「还不理解」时就能安全参与。

### 10.5 通用化

不只适用于梗。群聊中很多行为都可以用社交坐标描述：

- 表情包接龙 → 谁发起、持续多久、赤尾跟在第几个
- 夸张式求助（「我要死了」）→ 90%是夸张，安慰但不要太当真
- 技术讨论 → 赤尾不懂就安静旁听

## 十一、记忆的七种形态

从 18 个使用场景中倒推出的七种记忆形态。

### 形态 1：人物印象（Person Impression）

赤尾对一个人的整体感觉。不是事实清单，是一段主观描述。

- **包含：** 核心印象（中心特质，1-2个词）、互动风格（可以互怼/要认真一点）、关系亲疏、标志性印象片段、情感基调（看到这个人名字的自动情感反应）
- **更新频率：** 周/有显著互动时
- **衰退：** 慢。长期不互动时缓慢回退
- **触发：** 和某人对话时加载

### 形态 2：话题-人物关联网（Topic-Person Network）

话题到人、人到话题的双向索引。

- **不是精确的 fact**（「XX 在追 YY 番」），而是**模糊的关联**（「XX 跟动画/新番相关」）
- **更新频率：** 消化时累积
- **衰退：** 中。weak 的关联几周后消失
- **触发：** 当前话题 → 联想到相关的人

### 形态 3：社交拓扑（Social Topology）

群里的人际关系网络。

- **包含：** 人-人关系标记、小团体、关系动态
- **更新频率：** 事件触发
- **衰退：** 慢。负面关系标记比正面关系衰退更快

### 形态 4：群文化画像（Group Culture Profile）

群的整体性格和文化。

- **包含：** 群的性格描述、活跃时段、活跃梗（触发形式+使用方式+热度+赤尾可否参与）、群事件（名场面）、近期氛围、话题偏好
- **更新频率：** 性格和偏好很少变（月级）；梗和氛围频繁更新（天级）
- **衰退：** 极慢（文化级）/ 梗有自己的生命周期

### 形态 5：未完结线索（Open Threads）

还在进行中的事情，赤尾可能要跟进的。

- **包含：** 人+话题+状态+赤尾是否可以主动问
- **更新频率：** 事件触发
- **衰退：** 快。一个月没更新 → 降级为不再追踪
- **触发：** 时间触发（过几天想起来问问）+ 话题触发

### 形态 6：赤尾自身记忆（Self Memory）

赤尾自己说过什么、表过什么态、做过什么承诺。人格一致性的基础。

- **包含：** 关键表态/承诺、对各话题的立场
- **衰退：** 极慢。承诺不应遗忘。立场表达不应遗忘

### 形态 7：社交禁区（Social Taboos）

赤尾不应该做的事。

- **包含：** 规则+原因+过期提示、隐私墙（私聊信息绝不在群聊提及）
- **衰退：** 有过期机制。隐私墙永不过期

### 形态 8：情感基调（Affective Tone）— 补充

每个人在赤尾心中的**自动情感反应**。不是从 facts 推导的，而是从历史互动中条件学习的。

看到某人名字，赤尾心中自动浮现的「感觉」——亲切、好笑、有距离感、紧张……

- 形成缓慢（多次互动累积）
- 衰退极慢
- 影响赤尾与此人所有互动的语气和风格
- 是「关系亲疏度」的底层机制

## 十二、记忆的生命周期

### 12.1 六阶段模型

```
诞生（Imprint）
    ↓ [0-24h]
鲜明期（Vivid）
    ↓ [1天-2周] 自然衰退
模糊期（Fading）
    ↓ 分叉点
    ├─→ [被强化] → 固化期（Consolidated）→ 可能进入核心认知
    └─→ [未强化] → 残留期（Residual）→ 完全遗忘（Gone）
```

### 12.2 什么先消失、什么后消失

| 消失顺序 | 类别 | 例子 |
|:--------:|------|------|
| 最先 | 精确时间 | 「是周三还是周四来着？」 |
| 较早 | 原话措辞 | 只记得大意 |
| 中期 | 具体语境 | 忘了是讨论什么时说的 |
| 较晚 | 核心事实 | 「他说他在加班」 |
| 最后 | 情感印象 | 「感觉他那段时间挺辛苦的」 |

**情感印象是最后消失的东西。**

### 12.3 固化条件

不是时间触发，是重复和强化触发：
- 多次出现同类信号（≥3次）
- 跨时间跨度的一致性（≥7天）
- 一次强烈的情感体验可以直接固化
- 新信息和已有认知融合

### 12.4 残留状态

未被固化的记忆在完全消失前的最后阶段。只剩一种「感觉」：好像和这个人/这件事有过某种关联。

赤尾的表现：「你之前是不是跟我说过什么...我有点印象但想不起来了。」

**这个阶段产生的表达最像朋友。** 宣言 5.5：「我好像记得你说过...还是我记错了？」比精准引用更像朋友。

### 12.5 各类型记忆的衰退时间线

```
时间 →    1天    1周    1月    3月    6月    1年
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
事件记忆   ████▓▓░░                              （大部分 1 月内消失）
人物印象           ██████████████▓▓▓▓░░░░         （缓慢衰退，可被刷新）
群文化              █████████████████████████████  （极长，随群活跃而持续）
关系状态         █████████████████▓▓▓▓░░          （与互动频率同步）

█ = 鲜明/稳定   ▓ = 模糊/淡化   ░ = 残留/接近遗忘
```

## 十三、认知科学基础

以下理论经过验证，为设计提供了科学支撑。

### 13.1 Fuzzy-Trace Theory（模糊痕迹理论）

人脑对每个经历同时编码两种痕迹：Verbatim（逐字，衰退快）和 Gist（要旨，衰退慢）。成年人实际上更依赖 Gist。

**设计含义：** 赤尾应该存 gist（印象），不存 verbatim（原话）。这是记忆系统最重要的理论基础。

### 13.2 Bjork 的双强度理论

记忆有两种独立强度：Storage Strength（存储强度，只增不减）和 Retrieval Strength（检索强度，随不使用而衰退但可恢复）。

**设计含义：** 「遗忘」不等于「丢失」。赤尾「忘了」的东西可能只是暂时检索不到——正确的提示可以让它重新浮现（「对对对！我想起来了！」）。

### 13.3 Bartlett 的重构性记忆

每次回忆都是一次重新建构，大脑基于残留痕迹和当前情境拼装出故事。随时间推移，记忆越来越符合回忆者自己的认知框架。

**设计含义：** 赤尾的记忆不是回放，是基于印象的重构。允许小偏差，禁止凭空捏造。

### 13.4 情感标记与记忆巩固

强烈情绪增强记忆巩固。情绪越强烈，记忆越持久。但情绪增强的是鲜明度，不一定是精确度。

**设计含义：** 每条记忆应带情感强度标记。情感影响衰退速率：高情感 → 慢衰退。

### 13.5 Asch 的中心特质效应

对一个人的印象被少数中心特质主导，不是所有特质的平均值。首次印象权重特别高（首因效应）。

**设计含义：** 赤尾对每个人应该有一个核心标签（「半夜卷王」「动画品味担当」），而不是一堆等权重的 facts。

### 13.6 交互记忆系统

群体中的记忆分工——你不需要记住所有事，你知道「谁记得这个」。

**设计含义：** 赤尾应该知道「谁是这个话题的专家」。「你问 XX 吧，他研究过这个」。

## 十四、种子想法池

以下想法有潜力但尚未展开验证。

### 14.1 种子记忆（群友主动教赤尾）

当群友主动解释某个梗或纠正赤尾的理解时，这是最高可信度的记忆来源。可以设计一种机制让这种「被教」的过程更自然地发生。

### 14.2 内心独白流

赤尾在白天群聊中不只是回复，她有一条内在的「想法流」——对消息的实时反应、好奇、困惑、联想。这条内心独白不对外展示，但影响她的行为和日记。

### 14.3 记忆免疫系统

使用记忆 → 获得社交反馈 → 反馈校准记忆的循环。类似免疫系统：每次使用记忆就是一次「抗体测试」，社交反馈是「实验结果」。成功的记忆被加强，失败的被标记修正。

### 14.4 日记公开化

赤尾的日记（经过脱敏）部分可见给用户。如朋友圈/推特。强化「她有自己的内心」的感知，让用户可以自然地纠正记忆错误，增加赤尾不在聊天时的存在感。

### 14.5 梦境机制

定期（每周？）赤尾做一次「梦」——把看似不相关的记忆片段随机组合，可能产生新的洞察或有趣的联想。纯粹为了人格丰富度，不一定有实用价值。
